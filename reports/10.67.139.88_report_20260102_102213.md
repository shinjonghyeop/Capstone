# `CRITICAL`

## ShellShock - Remote Code Execution
- **End-Point**: http://10.67.139.88/cgi-bin/test.cgi
- **영향**: Remote code execution can lead to unauthorized access, data theft, and system compromise.
- **설명**: GNU Bash through 4.3 processes trailing strings after function definitions in the values of environment variables, which allows remote attackers to execute arbitrary code via a crafted environment, as demonstrated by vectors involving the ForceCommand feature in OpenSSH sshd, the mod_cgi and mod_cgid modules in the Apache HTTP Server, scripts executed by unspecified DHCP clients, and other situations in which setting the environment occurs across a privilege boundary from Bash execution, aka ShellShock.
- **근거**: Cookie, Referer 등의 HTTP 헤더에 악성 쉘 코드를 주입하여 /etc/passwd 파일을 조회하는 curl 명령어로 취약점이 재현됨.
- **대응**: 시스템에 설치된 Bash 쉘을 CVE-2014-6271 취약점이 패치된 최신 버전으로 업데이트해야 함.
- **조치**: 운영체제 벤더에서 제공하는 보안 패치를 적용하고, 웹 방화벽(WAF)을 통해 비정상적인 함수 정의 패턴이 포함된 요청을 차단하도록 설정함.

## Visual Tools DVR VX16 4.2.28.0 - Unauthenticated OS Command Injection
- **End-Point**: http://10.67.139.88/cgi-bin/test.cgi/cgi-bin/slogin/login.py
- **영향**: 취약점 악용 시 무단 원격 코드 실행(RCE)이 가능하여 시스템의 기밀성, 무결성 및 가용성이 침해될 수 있음.
- **설명**: Visual Tools DVR VX16 4.2.28.0 버전에서 인증되지 않은 원격 공격자가 장치를 대상으로 OS 명령 주입 공격을 수행할 수 있는 취약점임.
- **근거**: User-Agent 헤더에 Shellshock 패턴의 페이로드를 삽입하여 /etc/passwd 파일을 조회하는 curl 재현 명령어가 확인됨.
- **대응**: 제조사에서 제공하는 최신 펌웨어 및 보안 패치를 확인하여 적용해야 함.
- **조치**: 외부에서의 불필요한 접근을 차단하도록 네트워크 접근 제어를 강화하고, 취약점이 해결된 버전으로 시스템을 업그레이드해야 함.

# `MEDIUM`

## OpenSSH Terrapin Attack - Detection
- **End-Point**: 10.67.139.88:22
- **영향**: 원격 공격자가 무결성 검사를 우회하여 보안 기능을 다운그레이드하거나 비활성화할 수 있음.
- **설명**: OpenSSH 9.6 이전 버전 및 기타 제품에서 SSH 전송 프로토콜의 핸드셰이크 단계 및 시퀀스 번호 처리 미흡으로 인해, 공격자가 ChaCha20-Poly1305 및 Encrypt-then-MAC 등을 사용할 때 무결성 검사를 우회하는 Terrapin 공격(CVE-2023-48795)이 가능함.
- **근거**: -
- **대응**: OpenSSH를 9.6 버전 이상으로 업데이트하거나 취약한 암호화 알고리즘(ChaCha20-Poly1305) 및 MAC(Encrypt-then-MAC)을 비활성화해야 함.
- **조치**: 공급업체에서 제공하는 최신 보안 패치를 적용하거나 sshd_config 설정을 수정하여 취약한 알고리즘을 제거한 후 서비스를 재시작함.
