## 주요 취약점

# `HIGH`

## Centos Web Panel 0.9.8.480 - Local File Inclusion  
- **End-Point**: http://192.168.64.2:9991/www/FileInclusion/pages/lvl2.php/admin/index.php, http://192.168.64.2:9991/www/FileInclusion/pages/lvl1.php/admin/index.php  
- **영향**:  
  - Successful exploitation allows attackers to read sensitive files on the server.  
- **설명**:  
  Centos Web Panel 0.9.8.480 contains a local file inclusion vulnerability (CVE-2018-18323) enabling unauthorized access to restricted files.  
- **근거**:  
  - 재현: `curl` 명령어를 통해 공격자가 직접 취약점을 재현할 수 있음이 확인됨.  
- **대응**:  
  - 파일 시스템에서 직접 경로 접근을 차단하는 보안 조치 적용  
- **조치**:  
  1. CentOS 버전을 최신으로 업데이트  
  2. 웹 패널 및 관련 스크립트 파일의 권한 제한

# `MEDIUM`

## Fuzzing Parameters - Cross-Site Scripting  
- **End-Point**: http://192.168.64.2:9991/www/FileInclusion/pages/lvl2.php/, http://192.168.64.2:9991/www/XSS/XSS_level2.php/ 외 2개  
- **영향**:  
- **설명**: Cross-site scripting 공격이 반사 파라미터를 통해 서버 응답에 스크립트가 포함되어 발생함을 확인.  
- **근거**:  
  재현: curl 명령어로 공격 가능성을 재현한 로그 증거 제공.  
- **대응**:  
- **조치**:  
  - 입력값 검증 및 이스케이프 처리 강화  
  - HTTP 헤더 및 쿠키 파라미터 필터링 적용

## Django Debug Page - Cross-Site Scripting  
- **End-Point**: http://192.168.64.2:9991/www/XSS/XSS_level1.php/create_user/  
- **영향**:  
  - Successful exploitation 가능 → 브라우저 내에서 임의 JavaScript 실행  
  - 세션 하이재킹, 페이지 변조, 정보 유출 등 심각한 보안 위협  
- **설명**: Django 1.10.x ~ 1.11.x에서 DEBUG 플래그 하에 HTML autoescaping이 비활성화된 템플릿을 통해 발생하는 취약점  
- **근거**:  
  - 재현 가능한 테스트 사례 제공 (curl 기반 공격)  
  - CVE cve-2017-12794에서 확인된 기술적 결함  
- **대응**:  
  - Django Debug Page 사용 중지  
  - DEBUG 플래그 비활성화 (권장)  
- **조치**:  
  - Django 버전을 1.11.6 이상으로 업데이트
