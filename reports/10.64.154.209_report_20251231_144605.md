## 주요 취약점

# `CRITICAL`
## ShellShock - Remote Code Execution
- **End-Point**: http://10.64.154.209/cgi-bin/test.cgi/cgi-bin/test
- **영향**: 원격 코드 실행으로 인한 무단 접근, 데이터 탈취 및 시스템 침해 가능성 존재
- **설명**: GNU Bash 4.3 이하 버전에서 환경 변수 처리 시 함수 정의 뒤 문자열을 실행하는 취약점
- **근거**: `curl` 요청 시 Cookie 헤더에 `() { ignored; };` 구문을 삽입하여 재현 확인
- **대응**: 운영체제 패키지 관리자를 통해 Bash 쉘을 최신 보안 패치 버전으로 업데이트
- **조치**: 외부 노출된 CGI 스크립트 점검 및 WAF를 통한 악성 패턴 차단 설정

## Visual Tools DVR VX16 4.2.28.0 - Unauthenticated OS Command Injection
- **End-Point**: http://10.64.154.209/cgi-bin/test.cgi/cgi-bin/slogin/login.py
- **영향**: 인증되지 않은 공격자가 원격에서 임의의 시스템 명령어를 실행하여 제어권 탈취 가능
- **설명**: Visual Tools DVR VX16의 특정 버전에서 발생하는 커맨드 인젝션 취약점
- **근거**: `curl` 요청 시 User-Agent 헤더에 `() { :; };` 페이로드를 삽입하여 재현 확인
- **대응**: 제조사에서 제공하는 최신 펌웨어로 즉시 업데이트 수행
- **조치**: 펌웨어 업데이트 전까지 해당 장비의 외부 네트워크 접근을 차단하거나 격리

## Spring Security BCrypt Truncation & ACL Bypass Detection
- **End-Point**: http://10.64.154.209/cgi-bin/test.cgi/admin/search
- **영향**: 비밀번호 검증 우회 및 ACL 설정 무력화로 인한 관리자 권한 접근 가능성
- **설명**: 긴 비밀번호 입력 시 BCrypt 잘림 현상과 ACL 우회를 통한 비인가 접근 시도 탐지
- **근거**: `curl` POST 요청으로 72자 이상의 긴 비밀번호 문자열을 전송하여 재현 확인
- **대응**: Spring Security 라이브러리를 취약점이 해결된 최신 버전으로 업데이트
- **조치**: 비밀번호 길이 제한 정책 적용 및 중요 엔드포인트에 대한 2차 인증 도입 검토

# `MEDIUM`

- **설명**: OpenSSH 9.6 이전 버전의 핸드셰이크 과정에서 발생하는 프로토콜 무결성 취약점
- **근거**: N/A (버전 기반 탐지 또는 상세 재현 정보 없음)
- **대응**: OpenSSH 서버를 9.6 버전 이상으로 업데이트하여 프로토콜 취약점 해결
- **조치**: 취약한 암호화 알고리즘(ChaCha20-Poly1305 등) 비활성화 및 설정 검토
