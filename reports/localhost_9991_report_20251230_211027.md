## 주요 취약점

# `HIGH`

## [Responsive FileManager <9.13.4 - Local File Inclusion]
- **End-Point**: /www/FileInclusion/pages/lvl1.php/filemanager/ajax_calls.php
- **영향**: An attacker can read sensitive files, potentially leading to unauthorized access.
- **설명**: Before v9.13.4, external input is used to construct a restricted pathname.
- **근거**: `curl -X 'GET' -d '' .../filemanager/ajax_calls.php ...`
- **대응**: Upgrade Responsive FileManager to version 9.13.4 or later.
- **조치**: Validate user inputs to ensure they reside within expected directories.

## [Centos Web Panel 0.9.8.480 - Local File Inclusion]
- **End-Point**: /www/FileInclusion/pages/lvl1.php/admin/index.php 외 1개
- **영향**: Exploitation allows reading sensitive files on the server.
- **설명**: Version 0.9.8.480 suffers from LFI, XSS, and RCE vulnerabilities.
- **근거**: `curl -X 'GET' .../admin/index.php ...`
- **대응**: Upgrade Centos Web Panel to a patched version immediately.
- **조치**: Implement strict file path validation and disable unnecessary file inclusion.

## [WordPress MDC YouTube Downloader 2.1.0 - Local File Inclusion]
- **End-Point**: .../wp-content/plugins/mdc-youtube-downloader/includes/download.php 외 1개
- **영향**: Unauthorized access to sensitive files and potential code execution.
- **설명**: Version 2.1.0 allows reading arbitrary files via full pathname in file param.
- **근거**: `curl -X 'GET' .../includes/download.php ...`
- **대응**: Update the plugin to the latest secure version or remove if unsupported.
- **조치**: Sanitize the 'file' parameter to prevent directory traversal sequences.

## [Dolibarr ERP/CRM 3.2 Alpha - Multiple Directory Traversal]
- **End-Point**: /www/FileInclusion/pages/lvl1.php/document.php
- **영향**: Reading arbitrary files and possibly executing arbitrary code.
- **설명**: Vulnerabilities in v3.2.0 Alpha allow access via '..' in file parameters.
- **근거**: `curl -X 'GET' .../document.php ...`
- **대응**: Upgrade Dolibarr ERP/CRM to a stable, secure release.
- **조치**: Restrict access to the document.php script and validate input paths.

## [Carel pCOWeb HVAC BACnet Gateway 2.1.0 - Path Traversal]

- **영향**: Unauthenticated arbitrary file disclosure via directory traversal.
- **설명**: Improper verification of 'file' parameter in logdownload.cgi allows traversal.
- **근거**: `curl -X 'GET' .../usr-cgi/logdownload.cgi ...`
- **대응**: Apply vendor patches for Carel pCOWeb firmware.
- **조치**: Implement input filtering to block traversal characters like '../'.

## [FHEM 6.0 - Local File Inclusion]
