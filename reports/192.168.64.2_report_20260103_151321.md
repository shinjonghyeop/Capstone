## 주요 취약점

# `HIGH`

## Centos Web Panel 0.9.8.480 - Local File Inclusion  
- **End-Point**: http://192.168.64.2:9991/www/FileInclusion/pages/lvl2.php/admin/index.php, http://192.168.64.2:9991/www/FileInclusion/pages/lvl1.php/admin/index.php  
- **영향**:  
  성공적인 공격 실행 시 서버에서 민감한 파일을 읽을 수 있음.  
- **설명**:  
  Centos Web Panel의 특정 버전(0.9.8.480)에서 Local File Inclusion(LFI) 취약점이 발견됨. 이는 사용자 입력을 기반으로 비인가된 파일을 접근할 수 있는 가능성을 내포함.  
- **근거**:  
  재현된 CURL 요청을 통해 공격 벡터 확인 가능. 경로 조작으로 `/etc/passwd`와 같은 시스템 파일 노출 가능성 입증됨. CVE-2018-18323 보고서 참조.  
- **대응**:  
  1. 서버 파일 시스템 권한 제한 (예: `chmod 500 /etc/passwd`)  
  2. 입력 검증 및 출력 인코딩 적용  
  3. 정기적인 패치 배포 및 CVE 영향 분석 수행  
- **조치**:  
   immediate patching을 권장하며, CVE 영향 범위 내 모든 CVE 해결 업데이트 적용. 개발 환경에서는 WAF(Web Application Firewall) 추가 고려.

# `MEDIUM`

## Fuzzing Parameters - Cross-Site Scripting  
- **End-Point**: http://192.168.64.2:9991/www/FileInclusion/pages/lvl2.php/, http://192.168.64.2:9991/www/XSS/XSS_level2.php/ 외 2개  
- **영향**: 
- **설명**: 서버 응답에 반사된 매개변수 값으로 인해 Cross-Site Scripting(XSS)이 발생함을 확인.  
- **근거**:  
  재현 예시:  
  ```markdown
  curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (X11; Linux x86_64; rv:1.9.7.20) Gecko/ Firefox/3.6.15' 'http://192.168.64.2:9991/www/FileInclusion/pages/lvl2.php/?order_id=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-order_id%27%29%3E&cookie=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-cookie%27%29%3E%29'
  ```  
- **대응**:  
  1. 입력값 검증 및 이스케이프 처리 강화  
  2. HTTP 헤더 파라미터에 보안 헤더(X-XSS-Protection 등) 적용  
  3. 서버 측 스크립트에서 출력 인코딩 확인  
- **조치**:  
  - 반사 공격 방어를 위한 입력값 필터링 추가

## Django Debug Page - Cross-Site Scripting  
- **End-Point**: http://192.168.64.2:9991/www/XSS/XSS_level1.php/create_user/  
- **영향**:  
  성공적인 공격 실행 시 피해자 브라우저 컨텍스트에서 임의 JavaScript 실행 가능, 세션 하이재킹, 페이지 변조, 민감 정보 탈취 등이 발생할 수 있음.  
- **설명**:  
  Django 1.10.x ~ 1.11.x에서 HTML autoescaping이 비활성화된 특정 템플릿을 통해 발생하며, DEBUG=True 환경에서만 취약점이 노출됨.  
- **근거**:  
  [재현 예시] curl command로 스크립트 실행 시도 가능 (위 참조).  
  CVE-2017-12794 공개 정보 확인 가능.  
- **대응**:  
  - Django 버전을 1.11.6 이상으로 업데이트  
  - debug=False 설정 적용  
  - HTTP 요청 헤더에 `X-Frame-Options: DENY` 추가  
- **조치**:  
  1. 영향이 있는 엔드포인트(XSS_level1.php) 즉시 제거 또는 수정  
  2. 정기적인 보안 패치 관리 체계 수립
