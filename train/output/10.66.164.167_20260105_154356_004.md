## MIME Type Confusion
- **End-Point**: `/news.php`, `/dashboard.php` 외 2개
- **영향**: 브라우저가 리소스의 MIME 타입을 임의로 추측(Sniffing)하여 실행함으로써 악성 스크립트가 실행되거나 보안 정책이 우회될 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 브라우저가 선언된 `Content-Type`과 다른 방식으로 데이터를 해석할 위험이 있음.
- **근거**: `curl "http://10.66.164.167/news.php"` 명령을 통한 점검 결과 응답 헤더에서 관련 보안 설정이 확인되지 않음.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 서버가 지정한 MIME 타입만을 따르도록 강제함.
- **조치**: 웹 서버(Apache, Nginx 등) 설정 파일이나 애플리케이션의 보안 미들웨어 설정을 통해 해당 헤더를 전역적으로 적용함.
