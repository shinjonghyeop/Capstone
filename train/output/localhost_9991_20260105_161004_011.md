## Basic XSS Prober - Cross-Site Scripting
- **End-Point**: `http://localhost:9991/www/XSS/XSS_level5.php/%61%27%22%3e%3c%69%6e%6a%65%63%74%61%62%6c%65%3e`
- **영향**: 취약한 엔드포인트를 통해 악성 스크립트가 실행될 경우 사용자의 세션 쿠키 탈취, 페이지 변조, 피싱 공격 등의 피해가 발생할 수 있음.
- **설명**: 일반적인 보안 테스트를 통해 크로스 사이트 스크립팅(XSS) 취약점이 발견되었으며, 해당 경로를 통해 전달된 페이로드가 적절한 검증 없이 브라우저에서 렌더링될 가능성이 있음.
- **근거**: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (X11; Linux i686) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/122.0.0.0 Safari/537.36' 'http://localhost:9991/www/XSS/XSS_level5.php/%61%27%22%3e%3c%69%6e%6a%65%63%74%61%62%6c%65%3e'` 요청을 통해 인코딩된 HTML 태그가 포함된 경로로 접근 시 취약점이 재현됨.
- **대응**: 사용자로부터 입력받는 모든 데이터와 URL 경로 파라미터에 대해 필터링을 수행하고, 출력 시 HTML 인코딩 처리를 적용해야 함.
- **조치**: 서버 측에서 특수 문자(`<`, `>`, `"`, `'` 등)를 HTML 엔티티로 변환하도록 코드를 수정하고, 브라우저 측 보안 강화를 위해 `Content-Security-Policy`(CSP)를 설정함.
