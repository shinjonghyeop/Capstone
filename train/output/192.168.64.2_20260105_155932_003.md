## Content Security Policy Configuration
- **End-Point**: /, /download 외 1개
- **영향**: 브라우저가 신뢰할 수 없는 외부 리소스를 로드하거나 악성 스크립트를 실행하는 것을 방지할 수 없어 교차 사이트 스크립팅(XSS) 및 데이터 주입 공격에 취약함.
- **설명**: 대상 URL의 응답 헤더에 콘텐츠 보안 정책(CSP)이 설정되어 있지 않아 클라이언트 측 보안 강화 기능이 활성화되지 않음.
- **근거**: `curl "http://192.168.64.2:5000/"` 실행 시 응답 헤더에서 CSP 관련 설정이 확인되지 않음.
- **대응**: 웹 애플리케이션의 모든 응답에 대해 적절한 `Content-Security-Policy` 헤더를 포함하도록 구성하여 허용된 출처의 리소스만 로드되도록 제한함.
- **조치**: 웹 서버 설정이나 소스 코드에서 `Content-Security-Policy: default-src 'self';`와 같은 보안 정책을 추가하고 서비스 요구 사항에 따라 허용 범위를 구체화함.
