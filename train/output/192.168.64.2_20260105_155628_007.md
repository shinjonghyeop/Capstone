## Unencrypted Channels
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 12개
- **영향**: 데이터가 암호화되지 않은 평문으로 전송되어 네트워크 도청을 통한 민감 정보 유출 및 중간자 공격(MITM)의 위험이 있음.
- **설명**: 해당 호스트에 HTTPS 리다이렉션 설정이 존재하지 않아 모든 HTTP 요청이 일반 텍스트로 처리됨.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 명령을 통해 HTTP 프로토콜로 암호화 없이 접근이 가능함을 확인함.
- **대응**: 모든 HTTP 요청을 HTTPS로 강제 리다이렉트하고 올바른 SSL/TLS 인증서를 적용하여 보안 채널을 구축해야 함.
- **조치**: 웹 서버 설정 파일(Nginx의 `return 301 https://$host$request_uri;` 또는 Apache의 `RewriteEngine On`)을 수정하여 모든 80번 포트 요청을 443번 포트로 전달하도록 구성함.
