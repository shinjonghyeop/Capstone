## Reflected Cross Site Scripting
- **End-Point**: `/vuln`
- **영향**: 공격자가 사용자의 브라우저 내에서 임의의 스크립트를 실행하여 세션 쿠키 탈취, 페이지 변조, 악성 사이트 리다이렉션 등의 공격을 수행할 수 있음.
- **설명**: `/vuln` 엔드포인트의 `param` 파라미터에 입력된 값이 서버 측에서 적절한 검증이나 이스케이프 처리 없이 사용자에게 다시 반사되어 반사형 크로스 사이트 스크립팅(Reflected XSS) 취약점이 발생함.
- **근거**: `param` 파라미터를 통해 스크립트 주입이 가능함을 확인하였으며, 재현 경로는 `curl "http://localhost:8000/vuln?param=%3CScRiPt%3Ealert%28%27wb2f9ho4r5%27%29%3C%2FsCrIpT%3E"`와 같음.
- **대응**: 사용자 입력값을 신뢰하지 않고 서버 측에서 출력 시 HTML 엔티티 인코딩을 적용하며, 브라우저 측 보안 강화를 위해 `Content-Security-Policy` 헤더를 설정함.
- **조치**: 사용자로부터 전달받은 `param` 데이터를 응답 본문에 포함할 때 `<`를 `&lt;`, `>`를 `&gt;` 등으로 변환하는 인코딩 로직을 적용하고, 최신 보안 프레임워크의 자동 이스케이프 기능을 활성화함.
