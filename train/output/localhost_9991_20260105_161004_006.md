## Content Security Policy Configuration
- **End-Point**: `/www/XSS/XSS_level2.php`, `/www/XSS/XSS_level1.php` 외 3개
- **영향**: 교차 사이트 스크립팅(XSS), 클릭재킹, 데이터 주입 공격 등 클라이언트 측을 대상으로 하는 보안 위협에 노출될 수 있음.
- **설명**: 해당 웹 페이지의 응답 헤더에 콘텐츠 보안 정책(CSP)이 설정되어 있지 않아, 브라우저가 신뢰하지 않는 스크립트나 리소스를 실행하는 것을 차단하지 못함.
- **근거**: `curl "http://localhost:9991/www/XSS/XSS_level2.php"` 명령어를 통한 응답 확인 시 CSP 관련 헤더가 누락됨을 확인.
- **대응**: 웹 서버 또는 애플리케이션의 HTTP 응답 헤더에 `Content-Security-Policy`를 추가하여 허용되는 리소스의 출처를 정의함.
- **조치**: `Content-Security-Policy: default-src 'self';`와 같이 보안성이 높은 정책을 적용하고, 필요한 경우 스크립트 및 스타일 출처를 세부적으로 제한함.
