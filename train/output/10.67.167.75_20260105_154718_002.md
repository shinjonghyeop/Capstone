## Content Security Policy Configuration
- **End-Point**: `/admin/`, `/` 외 1개
- **영향**: Content Security Policy(CSP)가 설정되어 있지 않아 교차 사이트 스크립팅(XSS), 클릭재킹, 데이터 주입 공격 등 클라이언트 측 보안 위협에 대한 방어 계층이 부재함.
- **설명**: 대상 URL(`http://10.67.167.75:8161/admin/`)의 응답 헤더에 CSP가 정의되지 않았으며, 이는 브라우저가 신뢰할 수 있는 리소스의 출처를 제한하지 못함을 의미함.
- **근거**: `curl "http://10.67.167.75:8161/admin/"` 실행 시 응답 헤더에서 `Content-Security-Policy` 항목이 확인되지 않음.
- **대응**: 웹 서버 또는 애플리케이션 설정에서 `Content-Security-Policy` HTTP 응답 헤더를 추가하여 허용된 도메인 및 리소스 유형을 정의해야 함.
- **조치**: 모든 응답 헤더에 `Content-Security-Policy: default-src 'self';`와 같은 보안 정책을 적용하여 인가되지 않은 외부 스크립트나 리소스의 실행을 차단함.
