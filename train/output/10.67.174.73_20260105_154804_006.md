## Secure Flag cookie
- **End-Point**: `/dashboard.php`, `/register.php` 외 3개
- **영향**: 암호화되지 않은 HTTP 통신을 통해 세션 쿠키가 전송될 수 있으며, 네트워크 스니핑을 통해 공격자가 사용자의 세션 정보를 탈취하고 세션 하이재킹을 수행할 위험이 있음.
- **설명**: `PHPSESSID` 쿠키에 `Secure` 속성이 설정되지 않아 보안 연결(HTTPS)이 아닌 일반 연결(HTTP)에서도 쿠키가 노출될 수 있는 상태임.
- **근거**: `curl "http://10.67.174.73/dashboard.php"` 명령 실행 시 응답 헤더의 `Set-Cookie` 설정에서 `Secure` 플래그가 누락된 것을 확인.
- **대응**: 모든 보안 관련 쿠키에 `Secure` 플래그를 설정하여 HTTPS 환경에서만 쿠키가 전송되도록 강제해야 함.
- **조치**: `php.ini` 설정에서 `session.cookie_secure = On`으로 설정하거나, 애플리케이션의 `session_set_cookie_params()` 함수 또는 `setcookie()` 함수 호출 시 `secure` 파라미터를 `true`로 설정함.
