## Content Security Policy Configuration
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 8개
- **영향**: 사이트 간 스크립팅(XSS), 클릭재킹, 데이터 주입 공격 등에 취약해질 수 있으며 브라우저가 악성 리소스 로드를 차단할 수 있는 보호 계층이 결여됨.
- **설명**: 대상 URL 호출 시 응답 헤더에 콘텐츠 보안 정책(CSP)이 설정되어 있지 않아 웹 브라우저가 실행 가능한 스크립트나 로드 가능한 리소스의 범위를 제한할 수 없음.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 명령 실행을 통해 응답 헤더 내 CSP 관련 설정이 누락되었음을 확인.
- **대응**: 웹 애플리케이션 서버 또는 웹 서버(Nginx, Apache 등) 설정에서 `Content-Security-Policy` HTTP 응답 헤더를 추가하여 허용된 출처의 리소스만 실행되도록 제한함.
- **조치**: 웹 서버 설정 파일에 `Content-Security-Policy: default-src 'self';`와 같이 서비스 요건에 맞는 보안 정책을 정의하여 적용하거나 애플리케이션 코드에서 관련 헤더를 전송하도록 구현함.
