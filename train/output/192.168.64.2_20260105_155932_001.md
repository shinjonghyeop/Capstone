## Insecure Deserialization via SQLi File Insert
- **End-Point**: `http://192.168.64.2:5000/download/nuclei-37nMfh96D9tz30k2nN3A7Iqsw7E.bin/37nMfh96D9tz30k2nN3A7Iqsw7E`, `http://192.168.64.2:5000/download/nuclei-37nNUPQ5lRQoLVUktCG8wN7fyhV.bin/37nNUPQ5lRQoLVUktCG8wN7fyhV`
- **영향**: 원격 코드 실행(RCE)을 통한 시스템 권한 탈취 및 데이터베이스 조작 가능성
- **설명**: Stacked SQL Injection 취약점을 악용하여 데이터베이스에 악성 Pickle 객체를 삽입한 후, 다운로드 시점에 해당 데이터를 역직렬화하여 임의의 코드를 실행함
- **근거**: 재현 단계에서 다음 명령어를 통해 엔드포인트 호출 시 역직렬화 과정이 수행됨을 확인: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (X11; Linux x86_64; rv:1.9.5.20) Gecko/ Firefox/15.0' 'http://192.168.64.2:5000/download/nuclei-37nMfh96D9tz30k2nN3A7Iqsw7E.bin/37nMfh96D9tz30k2nN3A7Iqsw7E'`
- **대응**: SQL Injection 방지를 위해 매개변수화된 쿼리(Parameterized Query)를 사용하고, 외부에서 입력된 데이터의 역직렬화 수행을 금지해야 함
- **조치**: Python의 `pickle` 라이브러리 대신 `json`과 같은 안전한 직렬화 형식을 사용하고, 데이터베이스에 저장되는 데이터에 대한 무결성 검증 로직을 추가함
