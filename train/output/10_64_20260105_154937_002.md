## Content Security Policy Configuration
- **End-Point**: `/index.php`, `/`
- **영향**: CSP(Content Security Policy)가 부재할 경우 XSS(Cross-Site Scripting), 클릭재킹, 데이터 주입 공격 등 클라이언트 측 보안 위협에 노출될 위험이 높습니다.
- **설명**: `http://10.64.141.227/index.php`를 포함한 엔드포인트에 CSP 헤더가 설정되어 있지 않아, 브라우저가 악성 스크립트나 신뢰할 수 없는 외부 리소스의 실행을 차단하지 못하는 상태입니다.
- **근거**: `curl "http://10.64.141.227/index.php"` 명령을 통해 응답 헤더를 확인한 결과, `Content-Security-Policy` 설정이 누락되어 있음을 확인하였습니다.
- **대응**: 웹 애플리케이션의 응답 헤더에 `Content-Security-Policy`를 추가하고, 서비스에 필요한 최소한의 출처(Origin)만 허용하도록 정책을 수립해야 합니다.
- **조치**: 웹 서버(Nginx, Apache 등) 설정이나 애플리케이션 코드 내에서 `Content-Security-Policy: default-src 'self';`와 같은 보안 헤더를 전송하도록 구성하십시오.
