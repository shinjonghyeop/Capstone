## Unencrypted Channels
- **End-Point**: `/cgi-bin/test.cgi`
- **영향**: 암호화되지 않은 HTTP 통신을 사용함에 따라 네트워크 상에서 패킷 스니핑을 통한 데이터 탈취 및 중간자 공격(MitM)에 의한 정보 유출 위험이 있음.
- **설명**: 해당 호스트에 HTTPS 리다이렉션 설정이 되어 있지 않아 모든 HTTP 요청이 암호화되지 않은 평문(Clear Text) 상태로 전송됨.
- **근거**: `curl "http://10.64.162.78/cgi-bin/test.cgi"` 명령 실행 시 암호화되지 않은 채널로 응답이 수신됨을 확인.
- **대응**: 모든 HTTP 트래픽을 HTTPS로 강제 리다이렉션하고 SSL/TLS 인증서를 적용하여 통신 구간을 암호화해야 함.
- **조치**: 웹 서버 설정 파일에서 80번 포트 요청에 대해 443번 포트로의 301 리다이렉트를 설정하고 HSTS(HTTP Strict Transport Security) 헤더를 추가함.
