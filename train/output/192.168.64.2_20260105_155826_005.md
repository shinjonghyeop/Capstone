## Unencrypted Channels
- **End-Point**: `/`, `/login`
- **영향**: 네트워크 통신 내용이 암호화되지 않아 중간자 공격(MITM)을 통해 사용자의 로그인 정보나 민감한 데이터가 평문으로 노출될 위험이 있음.
- **설명**: 해당 호스트에 HTTPS 리다이렉션 설정이 적용되어 있지 않아 모든 HTTP 요청이 암호화되지 않은 평문(Clear Text) 상태로 통신됨.
- **근거**: `curl "http://192.168.64.2:5000/"` 명령어를 통한 재현 시 HTTPS로의 전환 없이 HTTP 요청이 그대로 처리됨을 확인.
- **대응**: 모든 HTTP 요청을 HTTPS로 강제 리다이렉션하고, HSTS(HTTP Strict Transport Security) 헤더를 설정하여 보안을 강화해야 함.
- **조치**: 웹 서버(Nginx, Apache 등) 설정에서 80 포트로 들어오는 요청을 443 포트로 리다이렉션하도록 구성하고 유효한 SSL/TLS 인증서를 설치함.
