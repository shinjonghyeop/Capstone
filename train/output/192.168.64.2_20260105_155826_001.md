## Cleartext Submission of Password
- **End-Point**: `/`
- **영향**: 비밀번호가 암호화되지 않은 평문으로 전송되어 네트워크 스니핑이나 중간자 공격(MITM)을 통해 사용자 계정 정보가 공격자에게 노출될 수 있음.
- **설명**: `http://192.168.64.2:5000/` URL에서 비밀번호 필드가 암호화되지 않은 상태로 전송되는 취약점이 발견됨.
- **근거**: `password` 필드 노출 확인 및 `curl "http://192.168.64.2:5000/"` 명령을 통한 재현 가능.
- **대응**: 모든 민감한 정보 전송 시 HTTPS(SSL/TLS) 프로토콜을 사용하여 통신 구간을 암호화해야 함.
- **조치**: 웹 서버에 SSL 인증서를 적용하고 HTTP 요청을 HTTPS로 강제 리다이렉트하며, 전송 방식 시 평문 노출을 방지하도록 설정을 변경함.
