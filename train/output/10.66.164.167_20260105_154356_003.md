## Clickjacking Protection
- **End-Point**: `/news.php`, `/dashboard.php` 외 2개
- **영향**: 공격자가 투명한 레이어를 생성하여 사용자의 클릭을 유도함으로써 의도하지 않은 설정 변경이나 데이터 삭제 등의 민감한 동작을 수행하게 할 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy`의 `frame-ancestors` 지시자가 설정되어 있지 않아 웹 사이트가 타 사이트의 프레임 내에 삽입될 수 있는 상태임.
- **근거**: `curl "http://10.66.164.167/news.php"` 실행 시 응답 헤더에서 `X-Frame-Options` 설정이 확인되지 않음.
- **대응**: 모든 HTTP 응답에 웹 페이지의 프레임 포함 여부를 제어하는 보안 헤더를 강제로 적용해야 함.
- **조치**: 웹 서버 또는 애플리케이션 설정에서 `X-Frame-Options: SAMEORIGIN` 또는 `Content-Security-Policy: frame-ancestors 'self'` 헤더를 추가함.
