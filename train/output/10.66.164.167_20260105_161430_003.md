## Clickjacking Protection
- **End-Point**: `/news.php`, `/dashboard.php` 외 2개
- **영향**: 공격자가 피해자의 웹 브라우저에 투명한 프레임을 씌워 의도하지 않은 클릭을 유도함으로써 민감한 기능을 실행하거나 정보를 변조할 수 있음.
- **설명**: 웹 애플리케이션의 HTTP 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy`의 `frame-ancestors` 지시자가 설정되어 있지 않아 발생함.
- **근거**: `curl "http://10.66.164.167/news.php"` 명령 실행 시 응답 헤더에서 프레임 제어 관련 보안 헤더가 확인되지 않음.
- **대응**: 모든 HTTP 응답에 대해 프레임 내 렌더링을 허용하지 않거나 동일 도메인 내에서만 허용하도록 보안 헤더 설정을 적용해야 함.
- **조치**: 웹 서버 또는 애플리케이션 설정에서 `X-Frame-Options: SAMEORIGIN` 헤더를 추가하거나, 최신 브라우저 대응을 위해 `Content-Security-Policy: frame-ancestors 'self'` 설정을 적용함.
