## Command execution
- **End-Point**: `/ping`
- **영향**: 서버 내 임의 명령 실행을 통한 시스템 권한 탈취 및 기밀 데이터 유출
- **설명**: `host` 파라미터에 전달되는 입력값에 대한 적절한 검증 절차가 없어 쉘 메타문자를 이용한 Command Injection이 가능함
- **근거**: `host` 파라미터에 `;env;` 페이로드 주입 시 시스템 환경 변수가 출력되는 것을 확인하였으며 재현 코드는 다음과 같음:
```bash
curl "http://host8.dreamhack.games:23482/ping" -e "http://host8.dreamhack.games:23482/ping" -d "host=%3Benv%3B"
```
- **대응**: 사용자 입력값에서 세미콜론(`;`), 파이프(`|`), 앰퍼샌드(`&`) 등 쉘 메타문자를 필터링하거나 차단
- **조치**: 시스템 명령어를 직접 호출하는 함수 대신 인자를 안전하게 분리하여 처리하는 API를 사용하고 화이트리스트 기반의 입력값 검증 로직을 적용하여 소스 코드 수정
