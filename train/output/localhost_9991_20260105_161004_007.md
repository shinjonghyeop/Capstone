## Clickjacking Protection
- **End-Point**: `/www/XSS/XSS_level2.php`, `/www/XSS/XSS_level1.php` 외 3개
- **영향**: 공격자가 투명한 레이어를 생성하여 사용자의 클릭을 유도함으로써 의도하지 않은 기능 실행, 설정 변경 또는 민감 정보 노출을 유발할 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options` 설정이 누락되어 있어 웹 페이지가 `<iframe>`, `<frame>`, `<object>` 등의 태그를 통해 외부 사이트에 삽입될 수 있는 취약점임.
- **근거**: `curl "http://localhost:9991/www/XSS/XSS_level2.php"` 명령을 통한 점검 시 응답 헤더에서 클릭재킹 방지를 위한 보안 헤더가 확인되지 않음.
- **대응**: 모든 HTTP 응답 헤더에 `X-Frame-Options`를 `DENY` 또는 `SAMEORIGIN`으로 설정하거나, `Content-Security-Policy`의 `frame-ancestors` 지시자를 사용하여 프레임 삽입 가능 범위를 제한해야 함.
- **조치**: 웹 서버 설정(Apache, Nginx 등) 또는 애플리케이션 필터에서 `X-Frame-Options: SAMEORIGIN` 헤더를 강제로 추가하도록 구성함.
