## Reflected Cross Site Scripting
- **End-Point**: `/www/XSS/XSS_level2.php`, `/www/XSS/XSS_level1.php` 외 2개
- **영향**: 공격자가 사용자의 브라우저에서 임의의 스크립트를 실행하여 세션 쿠키 탈취, 피싱 공격, 사용자 권한 오용 등을 수행할 수 있음.
- **설명**: `username` 파라미터를 통해 입력된 스크립트가 서버 측의 적절한 필터링 없이 사용자에게 다시 반사되어 실행되는 Reflected XSS 취약점이 존재함.
- **근거**: `username` 파라미터에 스크립트 주입 시 `alert` 함수 실행 확인. 재현: ```curl "http://localhost:9991/www/XSS/XSS_level2.php?username=%3CScRiPt%3Ealert%28%27wl466xp844%27%29%3C%2FsCrIpT%3E&submit=Submit" -e "http://localhost:9991/www/XSS/XSS_level2.php"```
- **대응**: 사용자 입력값이 페이지에 출력될 때 HTML 엔티티로 변환되도록 HTML 인코딩을 적용하고, 서버 측에서 입력값에 대한 화이트리스트 기반 검증을 수행해야 함.
- **조치**: `htmlspecialchars()`와 같은 보안 함수를 사용하여 특수 문자(`<`, `>`, `"`, `'`, `&`)를 변환하고, `Content-Security-Policy` 헤더를 설정하여 신뢰되지 않은 스크립트의 실행을 차단함.
