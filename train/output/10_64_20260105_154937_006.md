## Unencrypted Channels
- **End-Point**: `/index.php`, `/`
- **영향**: 암호화되지 않은 HTTP 통신을 통해 민감한 데이터(쿠키 등)가 평문으로 전송되어, 중간자 공격(MITM)을 통한 정보 유출 및 세션 탈취의 위험이 있음.
- **설명**: 서버가 HTTPS를 강제하지 않아 보안이 취약한 HTTP 연결을 통해 통신이 이루어지며, 응답 헤더 등에 포함된 민감한 정보가 암호화 없이 노출됨.
- **근거**: `curl "http://10.64.141.227/index.php"` 실행 시 암호화되지 않은 채널을 통해 민감한 쿠키 정보가 포함된 응답이 수신됨을 확인.
- **대응**: 모든 트래픽에 SSL/TLS 암호화를 적용하고, HTTP 요청을 HTTPS로 강제 리다이렉트하며 HSTS(HTTP Strict Transport Security)를 활성화해야 함.
- **조치**: 웹 서버 설정을 수정하여 80번 포트 접속 시 443번 포트로 리다이렉트하도록 구성하고, 쿠키 설정에 `Secure` 속성을 추가하여 암호화된 연결에서만 전송되도록 제한함.
