## Content Security Policy Configuration
- **End-Point**: `/`
- **영향**: CSP 헤더가 설정되지 않은 경우 사이트 간 스크립팅(XSS), 클릭재킹, 데이터 주입 등 다양한 클라이언트 측 공격에 대한 방어 계층이 누락되어 보안 위협에 노출될 수 있음.
- **설명**: 해당 URL(`http://192.168.64.2:5000/`)의 응답에 콘텐츠 보안 정책(CSP)이 설정되어 있지 않아, 브라우저가 실행 가능한 스크립트나 로드 가능한 리소스의 출처를 제한하지 못함.
- **근거**: `curl "http://192.168.64.2:5000/"` 명령을 통한 재현 결과, HTTP 응답 헤더에서 `Content-Security-Policy` 설정이 확인되지 않음.
- **대응**: 웹 서비스에서 사용하는 스크립트, 스타일, 이미지 등의 리소스 출처를 분석하여 신뢰할 수 있는 도메인 리스트를 기반으로 보안 정책을 수립해야 함.
- **조치**: 웹 서버 설정 또는 애플리케이션 응답 로직에 `Content-Security-Policy` HTTP 헤더를 추가하고, `default-src 'self'`와 같이 최소 권한 원칙을 적용한 정책을 명시함.
