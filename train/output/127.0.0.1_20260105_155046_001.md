## XSS Filter Bypass Detection (script/on/javascript)
- **End-Point**: `http://127.0.0.1:8000`, `http://127.0.0.1:8000/` 외 1개
- **영향**: 악성 스크립트 실행을 통해 사용자의 세션 쿠키 탈취, 페이지 변조, 피싱 공격 및 개인정보 유출이 발생할 수 있음.
- **설명**: "script", "on", "javascript:" 등 XSS 방어를 위한 주요 키워드 필터링을 우회하여 임의의 스크립트를 실행할 수 있는 취약점을 탐지함.
- **근거**: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (SS; Linux x86_64; rv:135.0) Gecko/20100101 Firefox/135.0' 'http://127.0.0.1:8000?q=<body+oonload=alert({{1}})>'` 명령어를 통해 `q` 파라미터에 필터링을 우회하는 페이로드 주입이 가능함을 확인.
- **대응**: 사용자 입력값에 대해 화이트리스트 기반의 검증을 수행하고, 웹 애플리케이션의 모든 출력 지점에서 적절한 HTML 엔티티 인코딩을 적용함.
- **조치**: 출력 시 `htmlspecialchars` 등의 보안 함수를 사용하여 특수 문자를 치환하고, `Content-Security-Policy (CSP)` 헤더를 설정하여 인라인 스크립트 및 원격 스크립트 실행을 제한함.
