## Clickjacking Protection
- **End-Point**: `/dashboard.php`, `/register.php` 외 5개
- **영향**: 공격자가 투명한 프레임을 통해 사용자의 클릭을 유도하여 의도하지 않은 기능을 실행하거나 설정을 변경하는 등의 조작이 가능함
- **설명**: HTTP 응답 헤더에 `X-Frame-Options`가 설정되어 있지 않아 해당 페이지가 다른 사이트의 `<iframe>`, `<frame>`, `<object>` 내에 삽입될 수 있는 취약점임
- **근거**: `curl "http://10.67.174.73/dashboard.php"` 명령을 통한 확인 결과 응답 헤더에 `X-Frame-Options` 설정이 누락됨
- **대응**: 모든 웹 페이지 응답에 적절한 `X-Frame-Options` 또는 `Content-Security-Policy: frame-ancestors` 헤더를 설정함
- **조치**: 웹 서버(Apache, Nginx 등) 설정이나 애플리케이션 코드에서 `X-Frame-Options: SAMEORIGIN` 또는 `X-Frame-Options: DENY` 헤더를 추가하도록 수정함
