## Information Disclosure - Full Path
- **End-Point**: `/memo`
- **영향**: 서버의 내부 디렉터리 구조가 노출되어 공격자가 시스템 환경을 파악하거나 추가적인 취약점 공격의 단서로 활용할 수 있음.
- **설명**: `/memo` 엔드포인트 호출 시 응답 본문에 `/etc/passwd`와 같은 시스템 경로 정보가 포함되어 반환되는 취약점임.
- **근거**: `curl "http://localhost:8000/memo?memo=hello"` 실행 시 응답에서 시스템 경로가 노출됨을 확인.
- **대응**: 사용자에게 반환되는 응답 데이터나 에러 메시지에서 서버의 내부 절대 경로가 포함되지 않도록 로직을 수정함.
- **조치**: 애플리케이션 설정에서 상세 에러 출력을 비활성화하고, 응답 값에서 `/etc/`와 같은 민감한 문자열을 필터링하거나 제거함.
