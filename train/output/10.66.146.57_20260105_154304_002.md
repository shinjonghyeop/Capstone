## Content Security Policy Configuration
- **End-Point**: `/note.txt`, `/index.php` 외 5개
- **영향**: XSS(Cross-Site Scripting), 클릭재킹, 데이터 인젝션 공격에 취약해질 수 있으며, 브라우저가 신뢰할 수 없는 외부 리소스를 로드하거나 실행하는 것을 차단하지 못함.
- **설명**: 대상 엔드포인트의 HTTP 응답 헤더에 Content-Security-Policy(CSP)가 설정되어 있지 않아 웹 사이트의 보안 정책을 브라우저에 전달하지 못하는 상태임.
- **근거**: `curl "http://10.66.146.57/note.txt"` 명령을 통한 확인 시 응답 헤더에 CSP 설정이 누락되어 있음.
- **대응**: 웹 애플리케이션의 보안 요구 사항에 따라 허용할 리소스의 출처를 정의하고 이를 CSP 헤더에 명시해야 함.
- **조치**: HTTP 응답 헤더에 `Content-Security-Policy`를 추가하고, `default-src 'self'`와 같이 신뢰 가능한 도메인 및 리소스만 허용하도록 설정함.
