## SQL Injection
- **End-Point**: `/www/SQL/sql1.php`
- **영향**: 데이터베이스 내 민감 정보 유출, 데이터 임의 조작 및 삭제, 관리자 권한 탈취 등의 심각한 보안 위협이 발생할 수 있음.
- **설명**: MySQL 데이터베이스 환경에서 `firstname` 파라미터를 통해 SQL 쿼리문을 주입할 수 있는 SQL Injection 취약점이 존재함.
- **근거**: `firstname` 파라미터에 특수 문자가 포함된 페이로드를 전송하여 취약 여부 확인.
`curl "http://192.168.64.2:9991/www/SQL/sql1.php" -e "http://192.168.64.2:9991/www/SQL/sql1.php" -d "firstname=default%C2%BF%27%22%28&submit=Submit"`
- **대응**: SQL 쿼리 작성 시 Prepared Statement를 사용하거나 Parameterized Query를 적용하여 사용자 입력값이 쿼리 구조를 변경하지 못하도록 설정해야 함.
- **조치**: 웹 애플리케이션 방화벽(WAF)에서 SQL Injection 패턴을 차단하도록 설정하고, 데이터베이스 계정의 권한을 최소화하며 입력값에 대한 화이트리스트 기반 검증을 수행함.
