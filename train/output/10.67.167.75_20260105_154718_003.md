## Clickjacking Protection
- **End-Point**: `/admin/`, `/` 외 1개
- **영향**: 공격자가 투명한 레이어를 사용하여 사용자의 클릭을 유도함으로써 의도하지 않은 기능 실행이나 민감한 정보 수정을 유발할 수 있음.
- **설명**: 웹 페이지의 HTTP 응답 헤더에 `X-Frame-Options`가 설정되어 있지 않아 타 사이트의 `iframe` 내에 해당 페이지가 삽입될 수 있는 취약점임.
- **근거**: `curl "http://10.67.167.75:8161/admin/"` 실행 시 응답 헤더에서 클릭재킹 방지를 위한 보안 헤더가 확인되지 않음.
- **대응**: 모든 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy`의 `frame-ancestors` 지시어를 설정하여 프레임 렌더링을 제한함.
- **조치**: 웹 서버 또는 애플리케이션 설정에서 `X-Frame-Options: SAMEORIGIN` 헤더를 추가하여 동일 도메인 내에서만 프레임 삽입이 가능하도록 조치함.
