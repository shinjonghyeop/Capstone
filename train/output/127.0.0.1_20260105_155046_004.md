## MIME Type Confusion
- **End-Point**: `/`, `/memo` 외 3개
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 결정하는 MIME Sniffing이 발생할 수 있으며, 이를 통해 공격자가 의도한 악성 스크립트가 실행되는 등 교차 사이트 스크립팅(XSS) 공격에 노출될 위험이 있음.
- **설명**: 서버 응답 헤더에 `X-Content-Type-Options`가 설정되어 있지 않아 브라우저가 `Content-Type` 헤더를 무시하고 데이터를 기반으로 형식을 추측하게 됨.
- **근거**: `curl "http://127.0.0.1:8000/"` 실행 결과 응답 헤더에서 `X-Content-Type-Options: nosniff` 설정이 확인되지 않음.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저의 MIME Sniffing 기능을 비활성화하고 선언된 `Content-Type`만 사용하도록 강제함.
- **조치**: 웹 서버(Nginx, Apache 등) 또는 애플리케이션 프레임워크의 설정에서 `X-Content-Type-Options` 헤더 값을 `nosniff`로 적용함.
