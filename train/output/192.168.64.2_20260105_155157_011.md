## Reflected Cross Site Scripting
- **End-Point**: `/www/FileUpload/fileupload1.php`, `/www/XSS/XSS_level2.php` 외 4개
- **영향**: 공격자가 사용자의 브라우저에서 임의의 JavaScript 코드를 실행하여 세션 쿠키 탈취, 페이지 변조, 피싱 공격 등을 수행할 수 있음.
- **설명**: `file` 파라미터를 통해 전달된 입력값이 서버 측에서 적절한 필터링이나 인코딩 없이 응답 페이지에 포함되어 발생하는 반사형 크로스 사이트 스크립팅 취약점임.
- **근거**: `file` 파라미터의 파일명에 스크립트 주입 시 동작 확인. 재현 코드: `curl "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -e "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -F "submit=Submit" -F "file=@your_local_file;filename=<ImG src=z oNeRror=alert('wq7z5o30by')>"`
- **대응**: 모든 사용자 입력값에 대해 신뢰할 수 없는 데이터로 간주하고 입력 단계에서 특수문자 필터링 및 허용된 형식만 수용하는 화이트리스트 검증을 수행해야 함.
- **조치**: 사용자 입력값을 HTML 출력 시 `htmlspecialchars()`와 같은 함수를 사용하여 HTML Entity Encoding을 적용하고, `Content-Security-Policy (CSP)` 헤더를 설정하여 스크립트 실행을 제한해야 함.
