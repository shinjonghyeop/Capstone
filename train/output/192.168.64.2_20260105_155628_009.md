## Path Traversal
- **End-Point**: `/www/FileUpload/fileupload1.php`, `/www/FileInclusion/pages/lvl1.php` 외 1개
- **영향**: 공격자가 서버의 임의 디렉토리에 접근하거나 파일을 업로드하여 시스템 정보를 유출하고 원격 코드 실행(RCE)을 수행할 수 있음.
- **설명**: `file` 파라미터를 통해 전달되는 입력값이 `move_uploaded_file()` 함수에서 적절히 검증되지 않아 발생하는 취약점으로, 경로 조작을 통해 의도하지 않은 위치에 파일 저장이 가능함.
- **근거**: `file` 파라미터를 대상으로 하며, 다음 명령어를 통해 재현 가능함: `curl "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -e "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -F "submit=Submit" -F "file=@your_local_file;filename=fileupload1.php"`
- **대응**: 파일 업로드 시 사용자 입력값에서 경로 순회 문자열(`../`, `..\`)을 필터링하고, 허용된 디렉토리 외부로의 파일 저장을 차단해야 함.
- **조치**: 서버 측에서 파일 이름을 난독화하여 재명명하고, 업로드 디렉토리의 실행 권한을 제거하며 화이트리스트 기반의 확장자 검증을 적용함.
