## MIME Type Confusion
- **End-Point**: `/dashboard.php`, `/register.php` 외 5개
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 결정하는 MIME Sniffing 공격에 노출될 수 있으며, 이는 교차 사이트 스크립팅(XSS) 등 추가적인 보안 위협으로 이어질 수 있음.
- **설명**: 서버 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 있어, 브라우저가 선언된 `Content-Type`을 무시하고 콘텐츠를 잘못된 형식으로 해석할 위험이 있음.
- **근거**: `curl "http://10.67.174.73/dashboard.php"` 실행 결과, 응답 헤더에서 `X-Content-Type-Options` 설정이 확인되지 않음.
- **대응**: 웹 서버 설정 또는 애플리케이션 코드에서 모든 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저의 MIME Sniffing 기능을 비활성화해야 함.
- **조치**: Nginx, Apache 등 웹 서버의 구성 파일이나 프레임워크의 보안 미들웨어를 사용하여 해당 보안 헤더를 전역적으로 적용하고, 리소스에 맞는 정확한 `Content-Type`이 전달되도록 설정함.
