## Unencrypted Channels
- **End-Point**: `/`, `/login` 외 4개
- **영향**: 데이터 전송 시 암호화가 이루어지지 않아 패킷 스니핑을 통한 민감 정보 유출 및 중간자 공격(MITM)에 노출될 수 있음.
- **설명**: 해당 호스트에 HTTPS 리다이렉션이 설정되어 있지 않아 모든 HTTP 요청이 암호화되지 않은 평문(Clear Text) 상태로 처리됨.
- **근거**: `curl "http://192.168.64.2:5000/"` 명령을 통해 HTTP 요청 시 암호화되지 않은 채널로 통신이 가능한 것을 확인.
- **대응**: 모든 HTTP 트래픽을 HTTPS로 리다이렉트하도록 설정하고 SSL/TLS 인증서를 적용하여 통신 과정을 암호화해야 함.
- **조치**: 웹 서버 설정에서 80번 포트 요청을 443번 포트로 리다이렉트하도록 구성하고, HSTS(HTTP Strict Transport Security) 헤더를 적용하여 보안 통신을 강제함.
