## Clickjacking Protection
- **End-Point**: `/note.txt`, `/index.php` 외 5개
- **영향**: 공격자가 웹 페이지를 투명한 프레임으로 삽입하여 사용자의 의도하지 않은 클릭을 유도하고, 권한 오용이나 데이터 수정 등의 민감한 동작을 수행하게 할 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options`가 설정되어 있지 않아, 브라우저가 해당 페이지를 `iframe`, `frame`, `object` 태그 내에 렌더링하는 것을 제한하지 않음.
- **근거**: `curl "http://10.66.146.57/note.txt"` 실행 결과 응답 헤더에서 `X-Frame-Options` 또는 `Content-Security-Policy: frame-ancestors` 설정이 누락된 것을 확인.
- **대응**: 웹 서버 또는 애플리케이션 설정에서 `X-Frame-Options` 헤더를 `DENY` 또는 `SAMEORIGIN`으로 설정하여 타 사이트에서의 프레임 삽입을 차단해야 함.
- **조치**: 웹 서버 설정 파일(예: Nginx의 `add_header X-Frame-Options "SAMEORIGIN";` 또는 Apache의 `Header always set X-Frame-Options "SAMEORIGIN"`)에 보안 헤더를 추가함.
