## Clickjacking Protection
- **End-Point**: `/index.php`, `/`
- **영향**: 공격자가 투명한 프레임을 사용하여 사용자의 클릭을 유도함으로써 의도하지 않은 기능 실행, 설정 변경 또는 정보 유출을 유발할 수 있음.
- **설명**: 웹 페이지의 HTTP 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy`의 `frame-ancestors` 지시자가 설정되어 있지 않아 외부 사이트에서 해당 페이지를 프레임으로 삽입할 수 있는 취약점임.
- **근거**: `curl "http://10.64.141.227/index.php"` 명령을 통해 응답 헤더를 확인한 결과 `X-Frame-Options` 설정이 존재하지 않음을 확인.
- **대응**: 모든 HTTP 응답 헤더에 클릭재킹 방지를 위한 보안 헤더를 설정하여 웹 페이지가 신뢰되지 않은 타 사이트의 프레임 내에 표시되지 않도록 제한해야 함.
- **조치**: 웹 서버 설정 또는 애플리케이션에서 `X-Frame-Options: SAMEORIGIN` 또는 `Content-Security-Policy: frame-ancestors 'self'` 헤더를 추가하도록 설정함.
