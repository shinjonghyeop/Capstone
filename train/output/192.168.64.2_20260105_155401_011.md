## Reflected Cross Site Scripting
- **End-Point**: `/www/FileUpload/fileupload1.php`, `/www/XSS/XSS_level2.php` 외 4개
- **영향**: 공격자가 사용자의 브라우저에서 임의의 스크립트를 실행하여 세션 쿠키 탈취, 개인정보 유출, 피싱 공격 등을 수행할 수 있습니다.
- **설명**: `file` 파라미터의 파일명(`filename`) 입력값에 대한 적절한 검증 및 출력 인코딩이 누락되어 발생하는 Reflected Cross Site Scripting 취약점입니다.
- **근거**: `file` 파라미터의 파일명에 스크립트 페이로드를 삽입하여 전송 시 스크립트가 실행됨을 확인하였습니다. 재현 코드: ```curl "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -e "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -F "submit=Submit" -F "file=@your_local_file;filename=<ImG src=z oNeRror=alert('wq7z5o30by')>"```
- **대응**: 사용자로부터 입력받는 모든 데이터에 대해 화이트리스트 기반의 검증을 수행하고, HTML 응답으로 출력 시 반드시 HTML 인코딩을 적용해야 합니다.
- **조치**: 파일 업로드 처리 소스코드에서 파일명에 포함된 `<, >, ", ', &` 등의 특수문자를 ```htmlspecialchars()```와 같은 함수를 사용하여 안전하게 변환하거나 필터링하도록 수정합니다.
