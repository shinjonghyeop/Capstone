## Unencrypted Channels
- **End-Point**: `/news.php`, `/dashboard.php` 외 1개
- **영향**: 암호화되지 않은 통신을 통해 세션 쿠키 등 민감한 정보가 노출되어 중간자 공격(MITM)에 의한 데이터 탈취 및 세션 하이재킹 위험이 발생할 수 있음.
- **설명**: `http://10.66.164.167/news.php`와 같은 경로에서 민감한 데이터(응답 내 쿠키)가 암호화되지 않은 HTTP 연결을 통해 전송됨. 서버가 리다이렉션을 수행하지만 HTTPS로 전환되지 않아 초기 데이터와 리다이렉션 과정이 그대로 노출됨.
- **근거**: `curl "http://10.66.164.167/news.php"` 실행 시 HTTP 프로토콜을 통해 민감한 정보가 포함된 응답이 전송됨을 확인.
- **대응**: 모든 HTTP 트래픽을 HTTPS로 강제 전환하고, HSTS(HTTP Strict Transport Security)를 설정하여 보안 연결을 유지해야 함.
- **조치**: 웹 서버(Nginx, Apache 등) 설정에서 80번 포트의 모든 요청을 443번 포트로 리다이렉트하도록 구성하고, 쿠키 설정 시 `Secure` 속성을 추가하여 HTTPS 환경에서만 전송되도록 제한함.
