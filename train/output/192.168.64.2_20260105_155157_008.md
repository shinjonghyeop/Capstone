## SQL Injection
- **End-Point**: `/www/SQL/sql1.php`
- **영향**: 데이터베이스 내 저장된 민감 정보 유출, 데이터 임의 변조 및 삭제, 인증 우회 및 데이터베이스 서버 제어권 획득 가능성.
- **설명**: `firstname` 파라미터를 통해 전달되는 입력값이 적절한 필터링 없이 SQL 쿼리에 직접 포함되어 MySQL 데이터베이스에 악의적인 쿼리를 실행할 수 있는 취약점임.
- **근거**: `firstname` 파라미터에 특수문자 조작을 통한 취약점 재현 확인. 재현 코드: `curl "http://192.168.64.2:9991/www/SQL/sql1.php" -e "http://192.168.64.2:9991/www/SQL/sql1.php" -d "firstname=default%C2%BF%27%22%28&submit=Submit"`
- **대응**: 사용자 입력값이 SQL 쿼리의 구조를 변경하지 못하도록 매개변수화된 쿼리(Parameterized Query)를 사용하고 입력값에 대한 검증 로직을 구현해야 함.
- **조치**: 소스 코드 내 SQL 실행 부분에서 `Prepared Statement`를 도입하여 쿼리와 데이터를 분리하고, 특수문자 및 예약어에 대한 필터링을 적용함.
