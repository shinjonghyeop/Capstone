## Cleartext Submission of Password
- **End-Point**: `/www/CommandExecution/CommandExec-1.php`
- **영향**: 네트워크 트래픽 스니핑이나 중간자 공격(MitM)을 통해 사용자의 비밀번호가 평문으로 노출되어 계정 탈취 및 민감 정보 유출의 위험이 있음.
- **설명**: `http://192.168.64.2:9991/www/CommandExecution/CommandExec-1.php` 경로에서 비밀번호 필드 데이터가 암호화되지 않은 평문 상태로 전송됨.
- **근거**: `curl "http://192.168.64.2:9991/www/CommandExecution/CommandExec-1.php"` 명령 실행 시 `password` 데이터가 평문으로 전송되는 것을 확인.
- **대응**: 전송 계층 보안(TLS/SSL)을 적용하여 HTTP 대신 HTTPS 프로토콜을 사용해 모든 통신 데이터를 암호화해야 함.
- **조치**: 서버에 SSL/TLS 인증서를 적용하고, 민감한 정보를 전송할 때는 반드시 HTTPS를 사용하도록 강제하며, 데이터 전송 시 URL 파라미터가 아닌 안전한 HTTP 메서드를 사용함.
