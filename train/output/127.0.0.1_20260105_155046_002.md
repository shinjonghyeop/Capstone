## Content Security Policy Configuration
- **End-Point**: `/`, `/memo` 외 3개
- **영향**: 콘텐츠 보안 정책(CSP)이 설정되지 않아 교차 사이트 스크립팅(XSS), 클릭재킹 및 데이터 주입 공격에 취약할 위험이 있음.
- **설명**: `http://127.0.0.1:8000/`를 포함한 엔드포인트 호출 시 `Content-Security-Policy` 응답 헤더가 설정되어 있지 않음.
- **근거**: `curl "http://127.0.0.1:8000/"` 호출 시 응답 헤더에서 CSP 관련 설정이 누락된 것을 확인함.
- **대응**: 웹 애플리케이션의 모든 응답에 적절한 보안 정책을 담은 `Content-Security-Policy` 헤더를 추가해야 함.
- **조치**: 웹 서버 또는 애플리케이션 설정에서 `Content-Security-Policy: default-src 'self';`와 같이 신뢰할 수 있는 리소스만 허용하는 정책을 적용함.
