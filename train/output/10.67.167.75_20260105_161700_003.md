## Clickjacking Protection
- **End-Point**: `/admin/`, `/` 외 1개
- **영향**: 공격자가 웹사이트를 투명한 `iframe`으로 삽입하여 사용자의 클릭을 유도함으로써, 사용자가 의도하지 않은 민감한 기능을 실행하거나 정보를 수정하게 할 수 있음.
- **설명**: `X-Frame-Options` HTTP 응답 헤더가 설정되지 않아 브라우저가 해당 페이지를 `frame`, `iframe`, `object` 등 내부에 렌더링하는 것을 제한하지 못함.
- **근거**: `curl "http://10.67.167.75:8161/admin/"` 명령을 통한 응답 확인 시 `X-Frame-Options` 헤더가 존재하지 않음을 확인함.
- **대응**: 모든 HTTP 응답 헤더에 프레임 렌더링 정책을 정의하여 신뢰하지 않는 도메인에서의 프레임 삽입을 차단해야 함.
- **조치**: 웹 서버 또는 애플리케이션 설정에서 `X-Frame-Options: SAMEORIGIN` 또는 `X-Frame-Options: DENY` 헤더를 추가하고, 최신 브라우저 지원을 위해 `Content-Security-Policy: frame-ancestors 'self'` 설정을 권고함.
