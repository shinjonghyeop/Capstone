## Content Security Policy Configuration
- **End-Point**: `/cgi-bin/test.cgi`, `/text/css` 외 3개
- **영향**: CSP가 설정되지 않은 경우 공격자가 악성 스크립트를 삽입하여 실행하는 XSS(Cross-Site Scripting), 데이터 주입, 클릭재킹 등의 클라이언트 측 공격에 취약해질 수 있음.
- **설명**: 웹 서버의 응답 헤더에 콘텐츠 보안 정책(Content Security Policy)이 구성되어 있지 않아 브라우저가 신뢰할 수 있는 리소스의 출처를 제한하지 못하는 상태임.
- **근거**: 다음의 명령어를 사용하여 해당 엔드포인트를 호출했을 때 응답 헤더에서 CSP 설정이 확인되지 않음: `curl "http://10.67.139.88/cgi-bin/test.cgi"`
- **대응**: 웹 애플리케이션 또는 웹 서버 설정에서 `Content-Security-Policy` HTTP 응답 헤더를 추가하여 허용 가능한 도메인과 리소스 로드 규칙을 정의함.
- **조치**: 보안 요구사항에 따라 `Content-Security-Policy: default-src 'self';`와 같은 정책을 기본적으로 적용하고, 외부 리소스 사용 시 해당 도메인을 화이트리스트에 추가하여 운영함.
