## Content Security Policy Configuration
- **End-Point**: `/dashboard.php`, `/register.php` 외 5개
- **영향**: Content Security Policy(CSP)가 설정되지 않아 사이트 간 스크립팅(XSS), 클릭재킹, 데이터 인젝션 등 클라이언트 측 보안 공격에 노출될 위험이 있으며, 공격자가 악성 스크립트를 실행하여 사용자 세션을 탈취하거나 정보를 유출할 수 있음.
- **설명**: 대상 엔드포인트의 HTTP 응답 헤더에 CSP 설정이 누락되어 있음. CSP는 웹 브라우저가 로드할 수 있는 리소스의 출처를 제한하여 악성 콘텐츠가 실행되는 것을 방지하는 보안 계층임.
- **근거**: `curl "http://10.67.174.73/dashboard.php"` 명령을 통해 응답 헤더를 확인한 결과 `Content-Security-Policy` 헤더가 존재하지 않음.
- **대응**: 웹 애플리케이션의 모든 응답 헤더에 적절한 CSP 정책을 추가하여 허용된 도메인에서만 스크립트, 스타일, 이미지 등의 리소스를 로드하도록 제한해야 함.
- **조치**: 웹 서버 설정 또는 애플리케이션 코드에서 `Content-Security-Policy: default-src 'self';`와 같은 보안 헤더를 추가하고, 서비스 요구 사항에 맞춰 `script-src`, `style-src` 등 세부 지시자를 정의함.
