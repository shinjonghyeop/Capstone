## Visual Tools DVR VX16 4.2.28.0 - Unauthenticated OS Command Injection
- **End-Point**: `http://10.64.162.78/cgi-bin/test.cgi/cgi-bin/slogin/login.py`
- **영향**: 성공적인 취약점 악용 시 인증되지 않은 원격 코드 실행(RCE)이 가능하며, 이는 대상 시스템의 기밀성, 무결성, 가용성을 완전히 손상시킬 수 있음.
- **설명**: Visual Tools DVR VX16 4.2.28.0 버전에서 발생하는 취약점으로, 인증되지 않은 원격 공격자가 특수하게 조작된 요청을 보내 대상 장치에서 OS 명령을 실행할 수 있음.
- **근거**: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Host: 10.64.162.78' -H 'User-Agent: () { :; }; echo ; echo ; /bin/cat /etc/passwd' 'http://10.64.162.78/cgi-bin/test.cgi/cgi-bin/slogin/login.py'` 명령을 통해 `User-Agent` 헤더에 주입된 쉘 명령어가 실행되어 `/etc/passwd` 파일 내용이 반환되는 것을 확인.
- **대응**: 제조사에서 제공하는 최신 보안 패치를 적용하여 취약점(CVE-2021-42071)을 해결하고, HTTP 요청 헤더 및 입력값에 대한 검증 로직을 강화함.
- **조치**: 취약한 버전의 소프트웨어를 즉시 업데이트하고, 패치 적용 전까지 외부 네트워크로부터의 불필요한 CGI 스크립트 접근을 방화벽 및 침입 차단 시스템(IPS)을 통해 제한함.
