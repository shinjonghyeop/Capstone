## MIME Type Confusion
- **End-Point**: `/cgi-bin/test.cgi`, `/text/css` 외 4개
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 결정(MIME Sniffing)하게 되며, 이를 통해 악성 스크립트가 실행되는 Cross-Site Scripting(XSS) 공격 등에 노출될 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`가 설정되어 있지 않아, 브라우저가 서버에서 지정한 `Content-Type`을 무시하고 데이터 형식을 추측할 수 있는 상태임.
- **근거**: `curl "http://10.67.139.88/cgi-bin/test.cgi"` 명령을 통한 점검 시 해당 보안 헤더가 누락된 것을 확인.
- **대응**: 모든 웹 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 MIME 타입을 강제로 추측하지 않도록 설정해야 함.
- **조치**: 웹 서버(Apache, Nginx 등) 설정 파일 또는 애플리케이션 프레임워크의 보안 필터에서 `X-Content-Type-Options` 헤더 값을 `nosniff`로 설정함.
