## Content Security Policy Configuration
- **End-Point**: `/cgi-bin/test.cgi`
- **영향**: CSP 보안 헤더가 부재할 경우 교차 사이트 스크립팅(XSS), 클릭재킹, 데이터 삽입 공격 등에 취약해지며 브라우저가 악성 스크립트 실행을 차단하지 못해 사용자 정보가 유출될 수 있음.
- **설명**: 해당 엔드포인트에서 Content-Security-Policy 헤더가 설정되지 않은 것이 확인됨. CSP는 웹 페이지에서 로드할 수 있는 리소스의 출처를 제한하여 보안을 강화하는 핵심적인 방어 메커니즘임.
- **근거**: `curl "http://10.64.162.78/cgi-bin/test.cgi"` 실행 시 응답 헤더에서 `Content-Security-Policy` 설정이 누락된 것을 확인함.
- **대응**: 웹 서버 구성이나 애플리케이션 응답 로직에서 서비스 운영에 필요한 최소한의 출처만 허용하는 적절한 CSP 정책을 수립하고 헤더를 추가해야 함.
- **조치**: HTTP 응답 헤더에 `Content-Security-Policy: default-src 'self';`와 같이 보안 정책을 명시하여 적용함.
