## MIME Type Confusion
- **End-Point**: `/www/XSS/XSS_level2.php`, `/www/XSS/XSS_level1.php` 외 3개
- **영향**: 브라우저가 응답 본문의 MIME 타입을 잘못 추측하여 악성 스크립트를 실행하는 등 XSS(Cross-Site Scripting) 공격에 취약해질 수 있음
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 브라우저의 MIME 스니핑 기능이 활성화된 상태임
- **근거**: `curl "http://localhost:9991/www/XSS/XSS_level2.php"` 실행 시 응답 헤더에서 `X-Content-Type-Options` 항목이 확인되지 않음
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 정의된 `Content-Type` 이외의 형식으로 해석하지 않도록 강제함
- **조치**: 웹 서버(Nginx, Apache 등) 환경 설정이나 애플리케이션의 공통 응답 로직에서 해당 보안 헤더를 일괄적으로 적용함
