## Clickjacking Protection
- **End-Point**: `/`, `/memo` 외 3개
- **영향**: 공격자가 구성한 악의적인 페이지 내에 대상 웹 사이트를 프레임으로 삽입하여, 사용자가 의도하지 않은 클릭을 유도하고 민감한 동작을 수행하게 하거나 정보를 유출시킬 수 있음.
- **설명**: 웹 애플리케이션의 HTTP 응답 헤더에 `X-Frame-Options`가 설정되어 있지 않아 브라우저가 해당 페이지를 `<iframe>`, `<frame>`, `<object>` 등의 태그 내에 렌더링하는 것을 제한하지 못하는 취약점임.
- **근거**: `curl "http://127.0.0.1:8000/"` 명령을 통해 응답 헤더를 확인한 결과, 프레임 삽입을 방지하기 위한 보안 헤더가 누락되어 있음을 확인함.
- **대응**: 웹 서버 설정 또는 애플리케이션 프레임워크 설정을 통해 모든 응답에 대해 적절한 보안 헤더를 강제 적용해야 함.
- **조치**: HTTP 응답 헤더에 `X-Frame-Options: DENY` 또는 `X-Frame-Options: SAMEORIGIN`을 설정하며, 최신 브라우저 지원을 위해 `Content-Security-Policy: frame-ancestors 'self'` 설정을 병행하여 적용함.
