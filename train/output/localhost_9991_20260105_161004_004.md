## Django Debug Page - Cross-Site Scripting
- **End-Point**: `http://localhost:9991/www/XSS/XSS_level1.php/create_user/`
- **영향**: 공격자가 피해자의 브라우저 환경에서 임의의 JavaScript 코드를 실행할 수 있으며, 이를 통해 세션 하이재킹, 웹사이트 변조, 민감한 정보 탈취 등이 가능함.
- **설명**: Django 1.10.x (1.10.8 미만) 및 1.11.x (1.11.5 미만) 버전에서 500 에러 디버그 페이지의 일부 템플릿에 HTML 자동 이스케이프가 비활성화되어 발생하는 취약점(CVE-2017-12794)임. `DEBUG=True` 설정 시 공격자가 조작된 요청을 통해 스크립트를 주입할 수 있음.
- **근거**: 다음 curl 명령어를 통해 `username` 파라미터에 주입된 스크립트가 실행됨을 확인: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 14_3) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.2 Safari/605.1.15' 'http://localhost:9991/www/XSS/XSS_level1.php/create_user/?username=%3Cscript%3Ealert(document.domain)%3C%2Fscript%3E'`
- **대응**: 운영 환경에서는 반드시 디버그 모드를 비활성화하고, 해당 취약점이 패치된 최신 버전의 Django로 업데이트해야 함.
- **조치**: Django 설정을 `DEBUG = False`로 변경하고, Django 버전을 1.10.8 또는 1.11.5 이상으로 업그레이드 수행.
