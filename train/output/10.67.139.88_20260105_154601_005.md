## Clickjacking Protection
- **End-Point**: `/cgi-bin/test.cgi`, `/text/css` 외 4개
- **영향**: 공격자가 투명한 프레임에 대상 웹 페이지를 삽입하여 사용자가 의도하지 않은 클릭을 유도함으로써 권한 남용이나 정보 탈취 등의 행위를 수행할 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options`가 설정되지 않아 웹 페이지가 `<iframe>`, `<frame>`, `<object>` 등의 태그 내에 포함될 수 있는 상태임.
- **근거**: `curl "http://10.67.139.88/cgi-bin/test.cgi"` 명령을 통한 응답 확인 시 `X-Frame-Options` 헤더가 누락된 것을 확인함.
- **대응**: 웹 서버나 애플리케이션의 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy: frame-ancestors`를 설정하여 프레임 삽입을 제한해야 함.
- **조치**: 모든 HTTP 응답 헤더에 `X-Frame-Options: SAMEORIGIN` 설정을 추가하여 동일한 도메인 내에서만 프레임 삽입이 가능하도록 조치함.
