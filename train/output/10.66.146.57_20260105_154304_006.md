## Secure Flag cookie
- **End-Point**: `/index.php`, `/logout.php` 외 4개
- **영향**: HTTPS가 아닌 암호화되지 않은 HTTP 연결을 통해 세션 쿠키가 전송될 수 있으며, 이를 통해 네트워크 스니핑 공격자에게 세션 정보가 탈취되어 세션 하이재킹이 발생할 수 있음.
- **설명**: `PHPSESSID` 쿠키에 `Secure` 속성이 설정되어 있지 않아 보안되지 않은 통신 채널을 통해서도 쿠키가 노출될 수 있는 상태임.
- **근거**: `curl "http://10.66.146.57/index.php"` 실행 결과, 응답 헤더의 `Set-Cookie` 설정에서 `Secure` 플래그가 누락된 것을 확인하였음.
- **대응**: 모든 세션 쿠키에 `Secure` 속성을 부여하여 HTTPS 프로토콜을 통해서만 쿠키가 전송되도록 강제해야 함.
- **조치**: `php.ini` 설정 파일에서 `session.cookie_secure = True`로 설정하거나, 애플리케이션 코드 내 `session_set_cookie_params(0, "/", "", true, true);`와 같이 `secure` 파라미터를 `true`로 설정하여 배포함.
