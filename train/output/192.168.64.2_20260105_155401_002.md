## Fuzzing Parameters - Cross-Site Scripting
- **End-Point**: `http://192.168.64.2:9991/www/XSS/XSS_level2.php/`, `http://192.168.64.2:9991/www/XSS/XSS_level1.php/` 외 1개
- **영향**: 공격자가 사용자의 브라우저에서 임의의 스크립트를 실행하여 세션 쿠키 탈취, 페이지 변조, 피싱 공격 등을 수행할 수 있음.
- **설명**: GET 요청 파라미터에 입력된 값이 서버 응답에 적절한 검증 없이 그대로 반사(Reflected)되어 나타나는 XSS 취약점이 발견됨.
- **근거**: 아래의 `curl` 명령어를 통해 다수의 파라미터(`location`, `uid`, `username` 등)에 삽입된 스크립트 페이로드가 실행됨을 확인.
```bash
curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_6) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/14.1.2 Safari/605.1.15' 'http://192.168.64.2:9991/www/XSS/XSS_level2.php/?location=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-location%27%29%3E&link_url=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-link_url%27%29%3E&post_mime_type=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-post_mime_type%27%29%3E&uid=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-uid%27%29%3E&host=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-host%27%29%3E&cmd=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-cmd%27%29%3E&link_id=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-link_id%27%29%3E&reset=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-reset%27%29%3E&nonce=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-nonce%27%29%3E&username=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-username%27%29%3E&site=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-site%27%29%3E&do=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-do%27%29%3E&email=%27%3E%22%3Csvg%2Fonload=confirm%28%27xss-email%27%29%3E'
```
- **대응**: 사용자로부터 입력받는 모든 파라미터에 대해 화이트리스트 기반의 검증을 수행하고 특수 문자를 필터링해야 함.
- **조치**: 웹 애플리케이션의 출력 단계에서 HTML 엔티티 인코딩(예: `htmlspecialchars()`)을 적용하고, `Content-Security-Policy` (CSP) 헤더를 설정하여 인라인 스크립트 실행을 차단해야 함.
