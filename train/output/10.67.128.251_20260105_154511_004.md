## Clickjacking Protection
- **End-Point**: `/text/css`, `//spip` 외 1개
- **영향**: 공격자가 웹 페이지를 투명한 프레임으로 씌워 사용자의 클릭을 유도함으로써 의도하지 않은 기능 실행, 설정 변경 또는 민감 정보 노출을 유발할 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy`의 `frame-ancestors` 지시자가 설정되어 있지 않아 해당 페이지가 다른 사이트의 프레임 내에 포함될 수 있는 취약점임.
- **근거**: `curl "http://10.67.128.251/text/css"` 실행 결과 응답 헤더에 클릭재킹 방지를 위한 보안 헤더가 누락된 것을 확인함.
- **대응**: 브라우저가 해당 페이지를 프레임(`<iframe>`, `<frame>` 등) 내에서 렌더링하는 것을 제한하도록 서버 설정을 변경해야 함.
- **조치**: 웹 서버 설정 또는 애플리케이션 코드에서 모든 응답에 `X-Frame-Options: SAMEORIGIN` 또는 `X-Frame-Options: DENY` 헤더를 추가하고, 최신 브라우저 지원을 위해 `Content-Security-Policy: frame-ancestors 'self'` 설정을 병행함.
