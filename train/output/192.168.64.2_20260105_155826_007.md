## Secure Flag cookie
- **End-Point**: `/login`
- **영향**: 암호화되지 않은 HTTP 연결을 통해 세션 쿠키가 전송될 수 있으며, 중간자 공격(MITM) 시 쿠키가 탈취되어 세션 하이재킹으로 이어질 수 있음.
- **설명**: `http://192.168.64.2:5000/login`에서 설정되는 `session` 쿠키에 `Secure` 속성이 설정되어 있지 않아 보안 취약점이 발생함.
- **근거**: `curl "http://192.168.64.2:5000/login" -e "http://192.168.64.2:5000/" -d "username=alice&password=Letm3in_"` 명령 실행을 통해 HTTP 환경에서 쿠키가 생성됨을 확인.
- **대응**: 쿠키 생성 시 `Secure` 플래그를 활성화하여 HTTPS 프로토콜을 통해서만 쿠키가 전송되도록 강제해야 함.
- **조치**: 애플리케이션 설정 또는 코드에서 쿠키 응답 헤더에 `Secure` 속성을 추가하고 서비스 전체에 SSL/TLS를 적용함.
