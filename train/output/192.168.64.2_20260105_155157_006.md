## MIME Type Confusion
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 8개
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 결정하는 MIME 스니핑을 허용하게 되며, 이는 공격자가 의도한 악성 스크립트 실행(XSS) 등의 보안 위협으로 이어질 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 있어, 웹 브라우저가 서버에서 지정한 Content-Type을 무시하고 콘텐츠를 해석할 수 있는 상태임.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 명령을 통한 확인 결과, 응답 헤더에 `X-Content-Type-Options` 설정이 포함되어 있지 않음.
- **대응**: 서버의 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 강제로 지정된 MIME 타입만을 사용하도록 설정해야 함.
- **조치**: 웹 서버(Nginx, Apache 등) 설정 파일이나 애플리케이션의 공통 보안 헤더 설정 로직에서 `X-Content-Type-Options` 헤더 값을 `nosniff`로 지정하여 배포함.
