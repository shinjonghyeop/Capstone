## Cleartext Submission of Password
- **End-Point**: `/`
- **영향**: 네트워크 트래픽을 가로채는 공격자가 사용자의 비밀번호를 평문으로 탈취하여 계정을 도용할 위험이 있음.
- **설명**: `http://192.168.64.2:5000/` 경로에서 비밀번호 필드(`password`)가 암호화되지 않은 평문 상태로 전송됨.
- **근거**: `curl "http://192.168.64.2:5000/"` 실행 시 `password` 정보가 평문으로 전송되는 것을 확인.
- **대응**: 전송 계층 보안(TLS/SSL)을 적용하여 데이터 전송 과정을 암호화해야 함.
- **조치**: 웹 서버에 SSL/TLS 인증서를 설치하고 HTTP 프로토콜 대신 HTTPS를 사용하도록 강제함.
