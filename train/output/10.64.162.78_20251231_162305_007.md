## MIME Type Confusion
- **End-Point**: `/cgi-bin/test.cgi`
- **영향**: 브라우저가 응답 본문의 내용을 잘못 해석하여 악성 스크립트를 실행(XSS)하거나 리소스를 오해석함으로써 보안 정책을 우회할 위험이 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`가 설정되어 있지 않아, 브라우저가 선언된 Content-Type을 무시하고 콘텐츠 내용을 기반으로 타입을 추측(MIME Sniffing)할 수 있는 취약점임.
- **근거**: 다음 명령어를 통해 해당 엔드포인트 호출 시 응답 헤더에서 `X-Content-Type-Options` 설정이 누락된 것을 확인 가능함. `curl "http://10.64.162.78/cgi-bin/test.cgi"`
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 강제로 적용하여 브라우저가 MIME 타입을 추측하지 않도록 설정함.
- **조치**: 웹 서버 설정(Nginx: `add_header X-Content-Type-Options nosniff;`, Apache: `Header set X-Content-Type-Options "nosniff"`) 또는 애플리케이션 코드 내에서 공통 헤더 추가 로직을 구현함.
