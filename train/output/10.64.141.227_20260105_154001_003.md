## Content Security Policy Configuration
- **End-Point**: `/index.php`, `/`
- **영향**: 사이트 간 스크립팅(XSS), 클릭재킹 및 기타 코드 주입 공격에 대한 방어 체계가 부재하여 사용자 데이터 탈취 및 악성 스크립트 실행 위험이 있음.
- **설명**: 해당 엔드포인트의 HTTP 응답 헤더에 콘텐츠 보안 정책(CSP)이 설정되어 있지 않아 브라우저가 신뢰할 수 있는 리소스와 그렇지 않은 리소스를 구분할 수 없음.
- **근거**: `curl "http://10.64.141.227/index.php"` 명령을 통한 확인 시 응답 헤더에 CSP 관련 항목이 존재하지 않음.
- **대응**: 웹 서버 설정 또는 애플리케이션 코드에서 `Content-Security-Policy` HTTP 응답 헤더를 추가하여 허용된 도메인 및 리소스 유형을 정의함.
- **조치**: 웹 서버(Nginx, Apache 등) 또는 프레임워크 설정에 `Content-Security-Policy: default-src 'self'; script-src 'self'; object-src 'none';`와 같은 보안 정책을 적용함.
