## Content Security Policy Configuration
- **End-Point**: `/`, `/text/css` 외 3개
- **영향**: 웹 브라우저에서 실행되는 스크립트 및 리소스 로드에 대한 통제가 불가능하여 XSS(Cross-Site Scripting), 클릭재킹, 데이터 인젝션 등의 공격에 노출될 수 있음.
- **설명**: 대상 URL인 `http://localhost:8000/`에 대해 콘텐츠 보안 정책(CSP)이 설정되어 있지 않아 브라우저가 신뢰할 수 없는 출처의 콘텐츠를 차단하지 못함.
- **근거**: `curl "http://localhost:8000/"` 명령을 통한 점검 시 응답 헤더에서 `Content-Security-Policy` 항목이 확인되지 않음.
- **대응**: 웹 서버 또는 애플리케이션 설정에서 `Content-Security-Policy` HTTP 응답 헤더를 정의하고 리소스별 허용 범위를 최소화하여 적용해야 함.
- **조치**: 응답 헤더에 `Content-Security-Policy: default-src 'self';`와 같은 정책을 추가하고 서비스 운영에 필요한 도메인만 선별적으로 허용하도록 구성함.
