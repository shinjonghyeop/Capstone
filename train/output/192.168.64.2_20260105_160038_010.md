## Secure Flag cookie
- **End-Point**: `/login`
- **영향**: HTTP 프로토콜을 통해 쿠키가 평문으로 전송될 수 있으며, 이는 중간자 공격(MITM)을 통한 세션 정보 탈취 및 계정 탈취로 이어질 수 있음.
- **설명**: `http://192.168.64.2:5000/login`에서 생성되는 `session` 쿠키에 `Secure` 플래그가 설정되어 있지 않아 보안 연결(HTTPS)이 아닌 환경에서도 쿠키가 전송됨.
- **근거**: `curl "http://192.168.64.2:5000/login" -e "http://192.168.64.2:5000/" -d "username=alice&password=Letm3in_"` 실행 결과, 응답 헤더의 `Set-Cookie` 설정에서 `Secure` 속성이 누락된 것을 확인.
- **대응**: 세션 쿠키를 포함하여 민감한 정보를 담은 모든 쿠키에 `Secure` 속성을 부여하여 HTTPS를 통해서만 전송되도록 강제해야 함.
- **조치**: 웹 애플리케이션 설정 또는 응답 헤더 생성 로직에서 쿠키 설정 시 `Secure` 플래그를 추가함. (예: `Set-Cookie: session=value; Secure; HttpOnly; SameSite=Lax`)
