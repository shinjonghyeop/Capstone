## MIME Type Confusion
- **End-Point**: `/news.php`, `/dashboard.php` 외 2개
- **영향**: 브라우저가 응답 본문의 내용을 잘못 해석하는 MIME Sniffing을 통해 악성 스크립트를 실행하거나 보안 정책을 우회할 수 있는 위험이 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 있어, 브라우저가 서버가 선언한 Content-Type을 무시하고 데이터를 임의의 형식으로 해석할 수 있음.
- **근거**: `curl "http://10.66.164.167/news.php"` 실행 시 응답 헤더에서 `X-Content-Type-Options` 설정이 확인되지 않음.
- **대응**: 웹 서버 또는 애플리케이션 설정에서 모든 HTTP 응답에 `X-Content-Type-Options: nosniff` 헤더를 포함하도록 구성해야 함.
- **조치**: Nginx, Apache 등 웹 서버의 설정 파일이나 프레임워크의 보안 미들웨어를 사용하여 해당 헤더를 전역적으로 적용함.
