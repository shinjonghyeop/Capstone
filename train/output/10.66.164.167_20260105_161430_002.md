## Content Security Policy Configuration
- **End-Point**: ` ```/news.php``` `, ` ```/dashboard.php``` ` 외 1개
- **영향**: 교차 사이트 스크립팅(XSS), 클릭재킹, 데이터 주입 등 악의적인 스크립트 실행 및 공격에 노출될 수 있음.
- **설명**: 해당 웹 페이지의 응답 헤더에 브라우저가 로드할 수 있는 리소스의 출처를 제어하는 Content Security Policy(CSP) 설정이 누락되어 있음.
- **근거**: ` ```curl "http://10.66.164.167/news.php"``` ` 명령 실행 시 응답 헤더에서 CSP 관련 설정이 확인되지 않음.
- **대응**: 신뢰할 수 있는 도메인 및 리소스만 실행 가능하도록 화이트리스트 기반의 보안 정책을 수립함.
- **조치**: 웹 서버 설정 또는 애플리케이션 응답 헤더에 ` ```Content-Security-Policy``` `를 추가하고 ` ```default-src 'self'``` `와 같이 적절한 보안 지시문을 적용함.
