## MIME Type Confusion
- **End-Point**: `/note.txt`, `/index.php` 외 5개
- **영향**: 브라우저가 응답 본문의 내용을 기반으로 MIME 타입을 추측(MIME Sniffing)하게 되어, 공격자가 업로드한 악성 스크립트가 실행되거나 보안 정책이 우회될 위험이 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`가 설정되어 있지 않아 브라우저가 선언된 `Content-Type`을 무시하고 데이터를 해석할 수 있는 취약점임.
- **근거**: `curl "http://10.66.146.57/note.txt"` 명령을 통해 확인 시 응답 헤더에 `X-Content-Type-Options` 설정이 누락되어 있음을 확인함.
- **대응**: 모든 웹 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 강제로 선언된 MIME 타입만 사용하도록 설정해야 함.
- **조치**: 웹 서버(Nginx, Apache 등) 환경 설정 파일이나 애플리케이션의 보안 미들웨어에서 `X-Content-Type-Options` 헤더를 `nosniff`로 적용함.
