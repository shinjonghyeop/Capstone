## Clickjacking Protection
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 8개
- **영향**: 공격자가 웹 사이트를 프레임 내에 삽입하여 사용자가 의도하지 않은 버튼을 클릭하게 유도함으로써 권한 오용이나 정보 유출 등의 피해를 입힐 수 있음.
- **설명**: HTTP 응답 헤더에 클릭재킹 공격을 방지하기 위한 `X-Frame-Options` 설정이 누락되어 있어 웹 페이지가 외부 프레임 내에서 렌더링될 수 있음.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 명령을 통한 확인 시 응답 헤더에 클릭재킹 방지 관련 헤더가 존재하지 않음.
- **대응**: HTTP 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy: frame-ancestors`를 설정하여 신뢰할 수 없는 도메인에서의 프레임 삽입을 차단해야 함.
- **조치**: 웹 서버 설정 또는 애플리케이션 코드에서 `X-Frame-Options: SAMEORIGIN` 또는 `X-Frame-Options: DENY` 헤더가 전송되도록 수정함.
