## Clickjacking Protection
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 12개
- **영향**: 공격자가 웹 페이지를 투명한 iframe으로 삽입하여 사용자가 의도하지 않은 클릭을 유도하고 민감한 기능을 실행하게 하는 클릭재킹 공격을 수행할 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options` 설정이 누락되어 있어 외부 사이트에서 해당 웹 페이지를 프레임 내에 포함시킬 수 있는 취약점임.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 명령을 통한 재현 확인 결과 응답 헤더에 보안 설정이 존재하지 않음.
- **대응**: 웹 서버 또는 애플리케이션 수준에서 프레임 렌더링 정책을 제어하는 보안 헤더를 설정해야 함.
- **조치**: 모든 응답 헤더에 `X-Frame-Options: SAMEORIGIN` 또는 `Content-Security-Policy: frame-ancestors 'self'`를 추가하여 동일한 출처에서만 프레임 삽입이 가능하도록 설정함.
