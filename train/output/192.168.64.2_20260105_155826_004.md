## MIME Type Confusion
- **End-Point**: `/`
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 추측하는 MIME 스니핑을 수행하며, 이로 인해 악성 스크립트 실행(XSS)이나 보안 정책 우회 공격이 발생할 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 있어, 브라우저가 서버가 선언한 `Content-Type`을 무시하고 데이터를 해석할 위험이 존재함.
- **근거**: `curl "http://192.168.64.2:5000/"` 호출 결과 응답 헤더에서 해당 보안 헤더가 누락된 것을 확인함.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 강제로 선언된 MIME 타입만 사용하도록 설정해야 함.
- **조치**: 웹 서버(Nginx, Apache 등) 설정 파일이나 어플리케이션의 보안 미들웨어 설정을 통해 해당 헤더를 일괄 적용함.
