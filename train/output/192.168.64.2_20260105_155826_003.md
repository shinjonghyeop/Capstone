## Clickjacking Protection
- **End-Point**: `/`
- **영향**: 공격자가 웹 페이지를 투명한 프레임으로 덮어씌워 사용자가 의도하지 않은 버튼 클릭이나 링크 선택을 유도할 수 있으며, 이를 통해 민감한 설정 변경이나 권한 남용 등의 행위가 발생할 수 있습니다.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options`가 설정되어 있지 않아 발생하는 취약점입니다. 이 헤더가 없으면 브라우저는 해당 페이지를 `<iframe>`, `<frame>`, `<object>` 등의 태그 내에 렌더링하는 것을 허용하므로 클릭재킹 공격에 노출됩니다.
- **근거**: `curl "http://192.168.64.2:5000/"` 실행 결과, 응답 헤더에서 클릭재킹 방지를 위한 `X-Frame-Options` 또는 `Content-Security-Policy: frame-ancestors` 설정이 확인되지 않았습니다.
- **대응**: 웹 서버나 애플리케이션 설정에서 `X-Frame-Options` 헤더를 추가하여 페이지가 프레임 내에서 렌더링되는 방식을 제어해야 합니다. 최신 브라우저를 위해 `Content-Security-Policy`의 `frame-ancestors` 지시문 사용도 권장됩니다.
- **조치**: HTTP 응답 헤더에 `X-Frame-Options: SAMEORIGIN` (동일 도메인 허용) 또는 `X-Frame-Options: DENY` (모든 프레임 거부)를 설정하여 외부 사이트에서의 임베딩을 차단합니다.
