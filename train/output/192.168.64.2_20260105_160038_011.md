## Reflected Cross Site Scripting
- **End-Point**: `/api/preview`
- **영향**: 공격자가 사용자의 브라우저에서 악성 스크립트를 실행하여 세션 쿠키 탈취, 페이지 변조, 피싱 공격 등을 수행할 수 있음.
- **설명**: `name` 파라미터에 전달된 입력값이 적절한 검증이나 인코딩 없이 응답 페이지에 포함되어 반환됨에 따라 발생하는 Reflected Cross Site Scripting 취약점임.
- **근거**: `name` 파라미터를 통해 주입된 스크립트가 실행됨을 확인: `curl "http://192.168.64.2:3000/api/preview?name=%22%3E%3C%2Fdiv%3E%3CScRiPt%3Ealert%28%27w60v5wullx%27%29%3C%2FsCrIpT%3E"`
- **대응**: 모든 사용자 입력값에 대해 HTML 엔티티 인코딩을 적용하고, 서버 측에서 허용된 문자열만 입력되도록 화이트리스트 기반 검증을 수행함.
- **조치**: 응답 본문에 `name` 파라미터 값을 출력하는 로직에 `htmlspecialchars`와 같은 인코딩 함수를 적용하고, `Content-Security-Policy` 헤더를 설정하여 스크립트 실행을 제한함.
