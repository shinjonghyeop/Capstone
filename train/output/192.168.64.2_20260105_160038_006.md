## Clickjacking Protection
- **End-Point**: `/`, `/hello` 외 3개
- **영향**: 공격자가 투명한 프레임을 웹 페이지 위에 겹쳐서 사용자가 의도하지 않은 버튼 클릭이나 링크 선택을 유도하여 민감한 기능을 실행하게 할 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options`가 설정되어 있지 않아 브라우저가 해당 페이지를 `<frame>`, `<iframe>`, `<object>` 등에서 렌더링하는 것을 제한하지 못함.
- **근거**: `curl "http://192.168.64.2:5000/"` 명령어를 통한 응답 확인 시 클릭재킹 방지를 위한 보안 헤더가 누락됨을 확인.
- **대응**: 웹 서버 설정이나 애플리케이션 소스 코드에서 HTTP 응답 헤더에 `X-Frame-Options: DENY` 또는 `X-Frame-Options: SAMEORIGIN`을 설정함.
- **조치**: 모든 웹 페이지 응답에 대해 `X-Frame-Options: DENY` 헤더를 추가하고, 최신 브라우저 호환성을 위해 `Content-Security-Policy: frame-ancestors 'none'` 설정을 병행함.
