## XSS Filter Bypass Detection (script/on/javascript)
- **End-Point**: `http://192.168.64.2:3000/`
- **영향**: 공격자가 사용자의 브라우저 내에서 임의의 악성 스크립트를 실행하여 세션 쿠키 탈취, 페이지 변조, 피싱 공격 등을 수행할 수 있습니다.
- **설명**: "script", "on", "javascript:"와 같은 일반적인 XSS 방어 키워드 필터링을 우회하는 패턴을 사용하여 크로스 사이트 스크립팅(XSS) 취약점을 유발하는 공격이 탐지되었습니다.
- **근거**: `q` 파라미터에 `oonnerror`와 같은 우회 키워드가 포함된 페이로드 `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (SS; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/142.0.0.0 Safari/537.36' 'http://192.168.64.2:3000/?q=<img+src=x+oonnerror="alert({{1}})">'` 삽입을 확인하였습니다.
- **대응**: 모든 사용자 입력값에 대해 HTML 엔티티 인코딩을 적용하고, 위험한 HTML 태그 및 속성을 제거하는 화이트리스트 기반의 필터링을 구현해야 합니다.
- **조치**: 출력 지점에서 데이터를 안전하게 렌더링하도록 수정하고, `Content-Security-Policy (CSP)` 헤더를 설정하여 인라인 스크립트 실행 및 신뢰되지 않은 도메인으로부터의 스크립트 로드를 차단합니다.
