## MIME Type Confusion
- **End-Point**: `/cgi-bin/test.cgi`
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 결정하는 MIME 스니핑을 수행할 수 있으며, 이를 통해 공격자가 악성 스크립트를 실행하거나 보안 정책을 우회할 수 있습니다.
- **설명**: 해당 엔드포인트의 응답 헤더에 `X-Content-Type-Options`가 설정되어 있지 않아 브라우저가 서버가 선언한 Content-Type을 무시하고 데이터를 해석할 위험이 있습니다.
- **근거**: `curl "http://10.64.162.78/cgi-bin/test.cgi"` 명령 실행 시 반환되는 응답 헤더에 `X-Content-Type-Options` 설정이 누락됨을 확인하였습니다.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 MIME 스니핑을 하지 않도록 강제해야 합니다.
- **조치**: 웹 서버(Apache, Nginx 등) 환경 설정 또는 애플리케이션 코드에서 `X-Content-Type-Options` 헤더를 `nosniff`로 설정하도록 조치하시기 바랍니다.
