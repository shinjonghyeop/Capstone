## MIME Type Confusion
- **End-Point**: `/index.php`, `/`
- **영향**: 브라우저가 응답 본문을 분석하여 MIME 타입을 임의로 추측하는 스니핑을 허용하며, 이는 XSS(Cross-Site Scripting) 등 악성 스크립트 실행으로 이어질 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 있어, 브라우저가 서버가 선언한 `Content-Type`을 무시하고 데이터를 해석할 위험이 있음.
- **근거**: `curl "http://10.64.141.227/index.php"` 명령을 통한 확인 시 응답 헤더에 관련 보안 설정이 존재하지 않음.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 지정된 MIME 타입만 따르도록 강제해야 함.
- **조치**: 웹 서버(Apache, Nginx 등) 설정 파일이나 애플리케이션 보안 미들웨어를 통해 `X-Content-Type-Options` 헤더를 적용함.
