## Clickjacking Protection
- **End-Point**: `/admin/`, `/text/css` 외 1개
- **영향**: 공격자가 웹 페이지를 투명한 프레임으로 씌워 사용자가 의도하지 않은 버튼을 클릭하게 함으로써 민감한 설정 변경이나 데이터 삭제 등의 행위를 유도할 수 있음.
- **설명**: HTTP 응답 헤더에 프레임 삽입을 제어하는 `X-Frame-Options`가 설정되어 있지 않아 발생하는 취약점으로, 악의적인 사이트에서 해당 페이지를 `<iframe>` 등으로 불러올 수 있음.
- **근거**: `curl "http://10.67.182.168:8161/admin/"` 명령을 통한 확인 결과, 응답 헤더에 `X-Frame-Options`가 포함되지 않음.
- **대응**: 웹 서버 또는 애플리케이션 설정에서 `X-Frame-Options` 헤더를 `DENY` 또는 `SAMEORIGIN`으로 설정하여 외부 사이트에서의 프레임 삽입을 방지함.
- **조치**: 웹 서버(Nginx, Apache 등)의 설정 파일에 `add_header X-Frame-Options "SAMEORIGIN";`과 같은 보안 헤더 설정 로직을 추가하고 적용함.
