## Clickjacking Protection
- **End-Point**: `/cgi-bin/test.cgi`
- **영향**: 공격자가 웹 페이지를 투명한 프레임으로 삽입하여 사용자가 의도하지 않은 버튼 클릭이나 민감한 기능을 실행하도록 유도하는 클릭재킹 공격이 가능합니다.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy`의 `frame-ancestors` 지시자가 설정되어 있지 않아 웹 브라우저가 해당 페이지를 프레임 내에 렌더링하는 것을 허용하고 있습니다.
- **근거**: `curl "http://10.64.162.78/cgi-bin/test.cgi"` 명령을 통한 응답 확인 시 보안 헤더가 누락되어 있음을 확인하였습니다.
- **대응**: 모든 HTTP 응답에 대해 프레임 렌더링 권한을 제어하는 보안 헤더를 추가하여 외부 사이트에서의 무단 프레임 삽입을 방지해야 합니다.
- **조치**: 서버 설정 또는 애플리케이션에서 `X-Frame-Options: SAMEORIGIN` 또는 `Content-Security-Policy: frame-ancestors 'self'` 헤더를 적용합니다.
