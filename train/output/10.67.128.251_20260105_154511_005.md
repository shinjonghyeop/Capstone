## MIME Type Confusion
- **End-Point**: `/text/css`, `//spip` 외 1개
- **영향**: 브라우저가 응답 본문의 MIME 타입을 잘못 해석하여 악성 스크립트를 실행(XSS)하거나 보안 정책을 우회할 수 있음
- **설명**: 웹 서버 응답에 `X-Content-Type-Options` 헤더가 설정되지 않아 브라우저가 콘텐츠 스니핑(Content Sniffing)을 통해 임의로 MIME 타입을 결정할 수 있는 상태임
- **근거**: `curl "http://10.67.128.251/text/css"` 명령을 통한 점검 결과 응답 헤더에 보안 설정이 누락된 것이 확인됨
- **대응**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 선언된 Content-Type만 사용하도록 강제함
- **조치**: 웹 서버 또는 애플리케이션 설정 파일에서 `X-Content-Type-Options` 보안 헤더를 `nosniff`로 일괄 적용함
