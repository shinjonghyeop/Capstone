## Content Security Policy Configuration
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 8개
- **영향**: 교차 사이트 스크립팅(XSS), 클릭재킹, 데이터 인젝션 공격에 노출될 수 있으며 악성 스크립트를 통한 사용자 정보 탈취 위험이 있음.
- **설명**: 해당 엔드포인트의 응답 헤더에 콘텐츠 보안 정책(CSP)이 설정되어 있지 않아 브라우저가 신뢰할 수 있는 리소스만 실행하도록 제한할 수 없음.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 명령 실행 시 응답 헤더에서 CSP 관련 설정이 확인되지 않음.
- **대응**: 웹 서버 설정 또는 애플리케이션 응답에 `Content-Security-Policy` 헤더를 추가하여 허용된 리소스 출처를 명시해야 함.
- **조치**: `Content-Security-Policy: default-src 'self';`와 같이 신뢰할 수 있는 도메인만 허용하는 정책을 적용하고 인라인 스크립트 사용을 제한함.
