## Command execution
- **End-Point**: `/ping`
- **영향**: 공격자가 서버 시스템 권한으로 임의 명령어를 실행하여 시스템을 장악하거나 내부 데이터를 유출할 수 있음.
- **설명**: `/ping` 엔드포인트의 `host` 파라미터에서 입력값에 대한 검증이 미흡하여 세미콜론(`;`) 등의 특수 문자를 이용한 Command Injection이 발생함.
- **근거**: `host` 파라미터에 `;env;` 페이로드를 주입하여 시스템의 환경 변수가 출력되는 것을 확인하였으며 재현 명령어는 다음과 같음. `curl "http://host3.dreamhack.games:21995/ping" -e "http://host3.dreamhack.games:21995/ping" -d "host=%3Benv%3B"`
- **대응**: 사용자 입력값에서 명령어 실행에 사용될 수 있는 특수 문자(`;`, `&`, `|`, `$`, `(`, `)` 등)를 차단하는 필터링을 적용하거나 화이트리스트 기반의 입력 값 검증을 수행해야 함.
- **조치**: 시스템 명령어를 직접 호출하는 함수(`os.system`, `exec` 등)의 사용을 지양하고, 인자를 안전하게 분리하여 처리하는 API를 사용하거나 입력값에 대한 이스케이프 처리를 반드시 수행해야 함.
