## SQL Injection
- **End-Point**: `/www/SQL/sql1.php`
- **영향**: 데이터베이스 내 저장된 민감 정보 유출, 데이터 임의 변조 및 삭제, 관리자 권한 획득 가능성
- **설명**: `firstname` 파라미터를 통해 MySQL 데이터베이스에 악의적인 SQL 구문을 삽입하여 실행할 수 있는 SQL Injection 취약점이 존재함
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php" -e "http://192.168.64.2:9991/www/SQL/sql1.php" -d "firstname=default%C2%BF%27%22%28&submit=Submit"` 명령 실행 시 `firstname` 파라미터에 특수 문자가 포함된 페이로드가 전달되어 취약함이 확인됨
- **대응**: 모든 사용자 입력값에 대해 화이트리스트 방식의 검증을 수행하고 SQL 특수 문자를 필터링함
- **조치**: `Prepared Statement`와 파라미터화된 쿼리(`Parameterized Query`)를 사용하여 사용자 입력값이 쿼리 구조를 변경하지 못하도록 소스 코드를 수정함
