## Secure Flag cookie
- **End-Point**: `/index.php`
- **영향**: 암호화되지 않은 HTTP 통신을 통해 세션 쿠키가 평문으로 전송될 수 있으며, 이는 네트워크 스니핑을 통한 세션 하이재킹 공격으로 이어질 수 있음.
- **설명**: `http://10.64.141.227/index.php` 경로에서 발행되는 `PHPSESSID` 쿠키에 `Secure` 속성이 설정되어 있지 않아 보안 연결(HTTPS)이 아닌 환경에서도 쿠키가 전송됨.
- **근거**: `curl "http://10.64.141.227/index.php"` 명령을 통해 응답 헤더를 확인한 결과, `Set-Cookie` 헤더에 `Secure` 플래그가 포함되지 않음을 확인.
- **대응**: 모든 세션 및 민감한 정보를 담은 쿠키에 `Secure` 속성을 추가하여 HTTPS 프로토콜을 통해서만 전송되도록 제한해야 함.
- **조치**: `php.ini` 설정에서 `session.cookie_secure = True`로 변경하거나, 애플리케이션 코드 내에서 `session_set_cookie_params` 함수의 `secure` 파라미터를 `true`로 설정함.
