## Secure Flag cookie
- **End-Point**: `/login`
- **영향**: 쿠키가 암호화되지 않은 HTTP 연결을 통해 전송될 수 있어, 네트워크 스니핑을 통해 공격자가 사용자의 세션 쿠키를 탈취하고 세션 하이재킹 공격을 수행할 수 있음.
- **설명**: `http://192.168.64.2:5000/login`에서 발급되는 `session` 쿠키에 `Secure` 속성이 설정되어 있지 않아 보안에 취약함.
- **근거**: `curl "http://192.168.64.2:5000/login" -e "http://192.168.64.2:5000/" -d "username=alice&password=Letm3in_"` 명령을 통한 재현 결과, 응답 헤더의 `Set-Cookie` 설정에서 `Secure` 플래그가 누락된 것을 확인.
- **대응**: 모든 민감한 쿠키, 특히 세션 쿠키를 생성할 때 `Secure` 속성을 명시적으로 설정하여 HTTPS 프로토콜을 통해서만 전송되도록 제한해야 함.
- **조치**: 웹 서버 또는 애플리케이션 프레임워크의 세션 설정에서 `secure` 옵션을 `true`로 변경하고, 서비스 전체에 SSL/TLS를 적용함.
