## MIME Type Confusion
- **End-Point**: `/`, `/hello` 외 3개
- **영향**: 브라우저가 콘텐츠를 잘못 해석하여 악성 스크립트를 실행하는 MIME 스니핑 공격 및 Cross-Site Scripting(XSS) 취약점에 노출될 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 브라우저가 서버에서 지정한 MIME 타입 대신 응답 본문 내용을 분석하여 임의로 해석할 위험이 있음.
- **근거**: `curl "http://192.168.64.2:5000/"` 명령 수행 결과 응답 헤더에 `X-Content-Type-Options` 설정이 존재하지 않음을 확인.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저의 MIME 스니핑 동작을 강제로 중단시켜야 함.
- **조치**: 웹 서버(Nginx, Apache 등)의 구성 파일이나 애플리케이션의 보안 미들웨어 설정에서 `X-Content-Type-Options` 헤더를 `nosniff`로 지정함.
