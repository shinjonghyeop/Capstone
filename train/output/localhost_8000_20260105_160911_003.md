## MIME Type Confusion
- **End-Point**: `/`, `/text/css` 외 3개
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 결정(MIME Sniffing)하게 되며, 이를 통해 공격자가 의도한 악성 스크립트가 실행되거나 보안 정책이 우회될 수 있음.
- **설명**: 서버 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 있어, 브라우저가 Content-Type 헤더에 정의된 타입 대신 실제 데이터 내용을 바탕으로 타입을 추측하는 취약점임.
- **근거**: `curl "http://localhost:8000/"` 실행 결과 응답 헤더에 `X-Content-Type-Options`가 설정되지 않은 것을 확인함.
- **대응**: 웹 서버 또는 애플리케이션의 모든 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저의 MIME Sniffing 기능을 비활성화해야 함.
- **조치**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 적용하고, 제공되는 모든 리소스에 대해 올바른 `Content-Type`이 전달되도록 설정함.
