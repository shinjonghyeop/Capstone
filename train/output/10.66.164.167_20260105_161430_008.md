## Secure Flag cookie
- **End-Point**: `/news.php`, `/dashboard.php`
- **영향**: 암호화되지 않은 HTTP 연결을 통해 세션 쿠키가 전송될 수 있으며, 이는 중간자 공격(MitM)에 의한 세션 하이재킹 및 사용자 계정 탈취로 이어질 수 있음.
- **설명**: `http://10.66.164.167/news.php`에서 생성되는 `PHPSESSID` 쿠키에 `Secure` 플래그가 설정되어 있지 않아 보안 연결(HTTPS)이 아닌 환경에서도 쿠키가 노출됨.
- **근거**: `curl "http://10.66.164.167/news.php"` 명령을 통한 재현 결과, 응답 헤더의 `Set-Cookie` 설정에서 `Secure` 속성이 누락된 것이 확인됨.
- **대응**: 웹 애플리케이션에서 발행하는 모든 민감한 쿠키에 `Secure` 속성을 추가하여 HTTPS 프로토콜을 통해서만 쿠키가 전송되도록 강제해야 함.
- **조치**: `php.ini` 설정에서 `session.cookie_secure = True`를 적용하거나, 소스 코드 내 쿠키 생성 함수 호출 시 `secure` 파라미터를 `true`로 명시함.
