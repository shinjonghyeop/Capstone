## Clickjacking Protection
- **End-Point**: `/`, `/text/css` 외 3개
- **영향**: 공격자가 웹 페이지를 투명한 프레임으로 삽입하여 사용자가 의도하지 않은 클릭을 유도하고 민감한 기능을 실행하거나 정보를 탈취할 수 있음
- **설명**: HTTP 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy`의 `frame-ancestors` 지시자가 설정되지 않아 웹 페이지가 다른 사이트의 프레임 내에 렌더링될 수 있는 취약점임
- **근거**: `curl "http://localhost:8000/"` 실행 결과 응답 헤더에서 클릭재킹 방지를 위한 보안 헤더가 누락된 것을 확인함
- **대응**: 웹 서버 또는 애플리케이션 설정에서 모든 HTTP 응답에 프레임 렌더링 권한을 제어하는 보안 헤더를 추가해야 함
- **조치**: HTTP 응답 헤더에 `X-Frame-Options: SAMEORIGIN` 또는 `Content-Security-Policy: frame-ancestors 'self'`를 설정하여 동일 도메인 외의 프레임 삽입을 차단함
