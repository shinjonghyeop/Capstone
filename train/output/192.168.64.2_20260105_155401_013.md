## Command execution
- **End-Point**: `/www/CommandExecution/CommandExec-1.php`, `/www/FileInclusion/pages/lvl1.php` 외 2개
- **영향**: 공격자가 서버 운영체제의 명령을 임의로 실행하여 시스템 권한을 획득하거나 내부 데이터를 유출 및 파괴할 수 있음.
- **설명**: `username` 파라미터에 대한 입력값 검증이 부재하여 공격자가 시스템 명령어를 주입하고 실행할 수 있는 취약점임.
- **근거**: `username` 파라미터에 `id` 명령어를 주입하여 시스템 정보가 출력됨을 확인하였으며 재현 코드는 다음과 같음: `curl "http://192.168.64.2:9991/www/CommandExecution/CommandExec-1.php?username=id&password=Letm3in_" -e "http://192.168.64.2:9991/www/CommandExecution/CommandExec-1.php"`
- **대응**: 외부 입력값이 시스템 명령문의 인자로 직접 사용되지 않도록 로직을 개선하고 위험한 함수 사용을 지양해야 함.
- **조치**: 입력값에 대해 허용된 문자열만 통과시키는 화이트리스트 기반 필터링을 적용하고, 시스템 명령 실행 대신 프로그래밍 언어에서 제공하는 API를 사용하도록 수정함.
