## Clickjacking Protection
- **End-Point**: `/cgi-bin/test.cgi`
- **영향**: 공격자가 투명한 프레임을 사용하여 사용자의 클릭을 유도함으로써 의도하지 않은 기능을 실행하거나 민감한 정보를 수정하게 할 수 있음.
- **설명**: 해당 엔드포인트의 응답 헤더에 클릭재킹 공격을 방지하기 위한 `X-Frame-Options` 또는 `Content-Security-Policy` 설정이 누락되어 있음.
- **근거**: `curl "http://10.64.162.78/cgi-bin/test.cgi"` 명령어를 통한 응답 확인 시 보안 헤더가 존재하지 않음.
- **대응**: 웹 서버 설정 또는 애플리케이션 응답 헤더에 프레임 전송을 제어하는 보안 헤더를 추가해야 함.
- **조치**: HTTP 응답 헤더에 `X-Frame-Options: SAMEORIGIN` 또는 `Content-Security-Policy: frame-ancestors 'self'`를 설정하여 외부 사이트에서의 프레임 삽입을 차단함.
