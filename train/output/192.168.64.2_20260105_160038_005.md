## Content Security Policy Configuration
- **End-Point**: `/`, `/hello` 외 3개
- **영향**: CSP(콘텐츠 보안 정책)가 구성되지 않은 경우, XSS(교차 사이트 스크립팅), 클릭재킹, 데이터 주입 공격 등 악성 스크립트 실행 및 리소스 변조 위험이 발생할 수 있습니다.
- **설명**: `http://192.168.64.2:5000/`를 포함한 여러 엔드포인트에서 웹 브라우저의 보안 기능을 강화하는 `Content-Security-Policy` HTTP 응답 헤더가 설정되어 있지 않습니다.
- **근거**: `curl "http://192.168.64.2:5000/"` 명령을 실행하여 응답 헤더를 확인한 결과, CSP 관련 설정이 누락되어 있음을 확인하였습니다.
- **대응**: 브라우저에서 신뢰할 수 있는 소스의 콘텐츠만 실행 및 로드되도록 정의한 `Content-Security-Policy` 헤더를 서버 응답에 추가해야 합니다.
- **조치**: 웹 서버 또는 애플리케이션 설정에서 `Content-Security-Policy` 헤더를 구성하고, `default-src 'self'`와 같이 서비스 환경에 적합한 보안 지시문을 적용하십시오.
