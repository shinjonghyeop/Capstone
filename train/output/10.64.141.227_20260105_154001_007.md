## Unencrypted Channels
- **End-Point**: `/index.php`, `/`
- **영향**: 암호화되지 않은 HTTP 통신을 통해 쿠키와 같은 민감 정보가 평문으로 노출되어 중간자 공격(MITM)에 의한 데이터 탈취 및 세션 하이재킹 위협이 있음.
- **설명**: 서버가 HTTPS 사용을 강제하지 않아 `http://10.64.141.227/index.php`로의 비암호화 연결이 허용되며, 응답 내 쿠키 등 민감 데이터가 보호되지 않은 상태로 전송됨.
- **근거**: `curl "http://10.64.141.227/index.php"` 명령 실행 시 암호화되지 않은 채널을 통해 데이터가 전송됨을 확인.
- **대응**: 모든 웹 트래픽에 SSL/TLS 암호화를 적용하고, HTTP로 접속하는 모든 요청을 HTTPS로 강제 리다이렉트하도록 설정함.
- **조치**: 웹 서버에 HSTS(HTTP Strict Transport Security) 헤더를 설정하고, 쿠키 생성 시 `Secure` 속성을 부여하여 암호화된 연결에서만 전송되도록 조치함.
