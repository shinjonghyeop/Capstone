## MIME Type Confusion
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 8개
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 결정하는 MIME Sniffing을 허용하게 되어, 공격자가 의도한 악성 스크립트가 실행될 위험이 있음.
- **설명**: 서버 응답 헤더에 `X-Content-Type-Options` 설정이 누락되어 있어, 브라우저가 선언된 `Content-Type`을 무시하고 데이터를 추측하여 처리할 수 있는 상태임.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 실행 시 응답 헤더에서 `X-Content-Type-Options: nosniff` 설정이 확인되지 않음.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저의 MIME Sniffing 기능을 비활성화해야 함.
- **조치**: 웹 서버(Nginx, Apache 등) 환경 설정 파일 또는 애플리케이션 프레임워크의 보안 설정에서 `X-Content-Type-Options` 헤더 값을 `nosniff`로 지정함.
