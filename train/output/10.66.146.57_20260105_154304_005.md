## HttpOnly Flag cookie
- **End-Point**: `/index.php`, `/logout.php` 외 4개
- **영향**: 크로스 사이트 스크립팅(XSS) 취약점이 존재할 경우, 공격자가 자바스크립트를 이용해 사용자의 세션 쿠키(`PHPSESSID`)를 탈취하고 세션을 하이재킹할 수 있음.
- **설명**: 애플리케이션에서 사용하는 세션 쿠키인 `PHPSESSID`에 `HttpOnly` 플래그가 설정되어 있지 않아 클라이언트 측 스크립트에서 쿠키 데이터에 접근이 가능한 상태임.
- **근거**: `curl "http://10.66.146.57/index.php"` 명령을 통한 응답 확인 시 `Set-Cookie` 헤더에 `HttpOnly` 속성이 누락됨을 확인함.
- **대응**: 세션 쿠키를 포함하여 인증 및 세션 관리에 사용되는 모든 쿠키에 `HttpOnly` 플래그를 설정하여 브라우저 스크립트의 접근을 차단해야 함.
- **조치**: `php.ini` 설정 파일에서 `session.cookie_httponly = On`으로 수정하거나, 애플리케이션 코드 내에서 `session_set_cookie_params` 함수의 `httponly` 파라미터를 `true`로 설정함.
