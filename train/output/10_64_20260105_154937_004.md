## MIME Type Confusion
- **End-Point**: `/index.php`, `/`
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 추측(Sniffing)함으로써, 공격자가 의도한 악성 스크립트가 실행되거나 보안 정책을 우회할 위험이 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options` 설정이 누락되어 있어, 브라우저가 서버가 선언한 Content-Type과 다른 방식으로 리소스를 해석할 수 있는 취약점임.
- **근거**: `curl "http://10.64.141.227/index.php"` 실행 시 응답 헤더에 `X-Content-Type-Options` 설정이 존재하지 않음을 확인함.
- **대응**: 웹 서버 또는 애플리케이션 설정에서 모든 응답에 `X-Content-Type-Options: nosniff` 헤더를 포함하도록 구성해야 함.
- **조치**: 서버 설정 파일(Nginx, Apache 등) 또는 애플리케이션의 보안 미들웨어를 수정하여 `X-Content-Type-Options: nosniff` 헤더를 강제로 적용함.
