## MIME Type Confusion
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 12개
- **영향**: 브라우저가 콘텐츠의 MIME 타입을 임의로 추측(MIME Sniffing)하게 되어, 공격자가 업로드한 악성 스크립트가 실행되거나 보안 정책이 우회될 위험이 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 있어, 서버가 선언한 Content-Type과 실제 데이터 형식이 다를 경우 브라우저가 이를 임의로 해석할 수 있음.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 명령을 통한 재현 시 응답 헤더에서 `X-Content-Type-Options` 항목이 확인되지 않음.
- **대응**: 웹 서버나 애플리케이션의 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정을 추가하여 브라우저의 MIME Sniffing 기능을 비활성화해야 함.
- **조치**: 웹 서버(Nginx, Apache 등) 설정 파일 또는 애플리케이션 전역 헤더 설정 로직에 `add_header X-Content-Type-Options "nosniff";`와 같은 구문을 추가함.
