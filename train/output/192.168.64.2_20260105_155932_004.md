## Clickjacking Protection
- **End-Point**: `/`, `/download` 외 1개
- **영향**: 공격자가 투명한 레이어를 사용하여 사용자가 의도하지 않은 클릭을 유도함으로써 권한 남용, 설정 변경 또는 민감 정보 노출 등의 피해를 입힐 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Frame-Options`가 설정되어 있지 않아 웹 페이지가 `<iframe>`, `<frame>`, `<object>` 등의 요소 내에 삽입될 수 있는 상태임.
- **근거**: `curl "http://192.168.64.2:5000/"` 명령을 통해 응답 헤더를 확인한 결과 `X-Frame-Options` 헤더가 존재하지 않음을 확인.
- **대응**: 웹 브라우저가 해당 페이지를 프레임 내에서 렌더링하는 것을 제어할 수 있도록 보안 헤더를 설정해야 함.
- **조치**: 서버 응답 헤더에 `X-Frame-Options: SAMEORIGIN` 또는 더 강력한 정책인 `X-Frame-Options: DENY`를 추가하고, 최신 브라우저 대응을 위해 `Content-Security-Policy: frame-ancestors 'self'` 설정을 병행함.
