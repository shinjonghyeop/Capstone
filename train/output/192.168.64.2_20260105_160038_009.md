## Inconsistent Redirection
- **End-Point**: `/login`
- **영향**: 리다이렉션 응답 본문에 포함된 HTML 요소가 사용자에게 노출되어 피싱 공격이나 의도치 않은 정보 유출의 위험이 발생할 수 있음.
- **설명**: HTTP 3xx 리다이렉션 응답 시 본문에 예상치 못한 HTML 링크나 폼 데이터가 포함되어 있는 취약점임.
- **근거**: `curl "http://192.168.64.2:5000/login" -e "http://192.168.64.2:5000/" -d "username=alice&password=Letm3in_"` 명령을 통해 3xx 응답 본문의 HTML 포함 여부 확인 가능.
- **대응**: 리다이렉션 응답 시 본문을 비우거나 표준에 부합하는 최소한의 텍스트만 포함하도록 수정해야 함.
- **조치**: 서버 환경 설정 또는 애플리케이션 프레임워크의 리다이렉트 로직을 검토하여 본문에 민감한 HTML 태그나 폼이 생성되지 않도록 설정함.
