## Reflected Cross Site Scripting
- **End-Point**: `/www/FileUpload/fileupload1.php`, `/www/XSS/XSS_level2.php` 외 4개
- **영향**: 공격자가 사용자의 브라우저에서 임의의 자바스크립트를 실행하여 세션 쿠키 탈취, 웹 페이지 변조, 피싱 공격 등을 수행할 수 있음.
- **설명**: `file` 파라미터의 파일명 항목을 통해 주입된 악성 스크립트가 서버 응답에 적절한 검증 없이 포함되어 발생하는 반사형 크로스 사이트 스크립팅 취약점임.
- **근거**: `file` 파라미터의 `filename` 속성에 스크립트를 삽입하여 재현함. `curl "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -e "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -F "submit=Submit" -F "file=@your_local_file;filename=<ImG src=z oNeRror=alert('wq7z5o30by')>"`
- **대응**: 사용자로부터 입력받는 모든 파라미터에 대해 유효성 검증을 실시하고, 출력 시 특수 문자를 HTML 엔티티로 변환하는 인코딩을 적용함.
- **조치**: 서버 측에서 파일명을 출력할 때 `htmlspecialchars` 등의 함수를 사용하여 HTML 인코딩을 수행하거나, 사용자 입력값을 직접 출력하지 않도록 로직을 수정함.
