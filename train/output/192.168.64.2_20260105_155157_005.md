## Clickjacking Protection
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 8개
- **영향**: 공격자가 웹 사이트를 투명한 프레임으로 감싸 다른 페이지 위에 겹쳐 놓음으로써 사용자가 의도하지 않은 클릭을 유도하고 민감한 기능을 실행하게 할 수 있음.
- **설명**: 응답 헤더에 `X-Frame-Options` 또는 `Content-Security-Policy`의 `frame-ancestors` 지시문이 설정되어 있지 않아 브라우저가 해당 페이지를 `<frame>`, `<iframe>`, `<embed>`, `<object>` 내에 렌더링하는 것을 제한하지 못함.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 실행 결과 응답 헤더에서 클릭재킹 방지를 위한 보안 헤더가 확인되지 않음.
- **대응**: 웹 서버 설정 또는 애플리케이션 코드에서 모든 응답에 대해 `X-Frame-Options` 헤더를 추가하여 프레임 렌더링 정책을 정의함.
- **조치**: 보안 요구 사항에 따라 응답 헤더에 `X-Frame-Options: SAMEORIGIN` 또는 `X-Frame-Options: DENY`를 설정하고, 최신 브라우저 지원을 위해 `Content-Security-Policy: frame-ancestors 'self'` 설정을 병행함.
