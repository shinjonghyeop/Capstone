## Content Security Policy Configuration
- **End-Point**: `/cgi-bin/test.cgi`
- **영향**: Content Security Policy(CSP)가 설정되어 있지 않으면 교차 사이트 스크립팅(XSS), 클릭재킹, 데이터 주입 공격 등에 취약해질 수 있으며, 공격자가 사용자의 브라우저에서 악성 스크립트를 실행하거나 민감한 정보를 탈취할 수 있음.
- **설명**: `http://10.64.162.78/cgi-bin/test.cgi` URL에 대해 CSP 헤더가 구성되어 있지 않음. CSP는 신뢰할 수 있는 콘텐츠 소스를 브라우저에 알림으로써 악성 리소스가 실행되는 것을 방지하는 보안 계층임.
- **근거**: `curl "http://10.64.162.78/cgi-bin/test.cgi"` 실행 결과 응답 헤더에 `Content-Security-Policy` 설정이 존재하지 않음을 확인.
- **대응**: 웹 애플리케이션의 리소스 로딩 요구 사항을 분석하여 적절한 CSP 정책을 수립하고, 웹 서버 또는 애플리케이션 레벨에서 응답 헤더에 이를 적용해야 함.
- **조치**: HTTP 응답 헤더에 `Content-Security-Policy`를 추가하고, `default-src 'self'`와 같이 허용된 출처의 리소스만 실행되도록 엄격한 정책을 설정함.
