## Content Security Policy Configuration
- **End-Point**: `/www/SQL/sql1.php`, `/www/FileUpload/fileupload1.php` 외 12개
- **영향**: 교차 사이트 스크립팅(XSS), 클릭재킹, 데이터 주입 공격 등에 대한 추가적인 방어 계층이 부재하여 웹 애플리케이션의 보안 취약성을 악용한 공격에 노출될 수 있음.
- **설명**: 지정된 URL들에 대해 웹 브라우저가 신뢰할 수 있는 콘텐츠 소스를 제한하는 보안 표준인 Content Security Policy(CSP) 헤더가 설정되어 있지 않음.
- **근거**: `curl "http://192.168.64.2:9991/www/SQL/sql1.php"` 명령을 통한 확인 시 응답 헤더에 CSP 설정이 포함되어 있지 않음.
- **대응**: 웹 서버 또는 애플리케이션 프레임워크 설정을 통해 적절한 정책을 담은 `Content-Security-Policy` HTTP 응답 헤더를 추가함.
- **조치**: 웹 서버 설정(Nginx, Apache 등)이나 애플리케이션 코드에 `Content-Security-Policy: default-src 'self';`와 같은 보안 지침을 적용하여 허용되지 않은 외부 리소스의 실행을 차단함.
