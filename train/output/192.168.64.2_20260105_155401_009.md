## Path Traversal
- **End-Point**: `/www/FileUpload/fileupload1.php`, `/www/FileInclusion/pages/lvl1.php` 외 1개
- **영향**: 공격자가 서버의 임의 경로에 파일을 업로드하여 시스템 권한을 획득하거나 중요 데이터를 변조 및 유출할 수 있음.
- **설명**: `file` 파라미터를 통해 `move_uploaded_file()` 함수 사용 시 경로 조작(Path Traversal) 공격이 가능하며, 파일 업로드 시 경로 제약 사항을 우회할 수 있는 취약점임.
- **근거**: `file` 파라미터를 대상으로 다음의 재현 코드를 실행하여 취약점 확인: `curl "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -e "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -F "submit=Submit" -F "file=@your_local_file;filename=fileupload1.php"`
- **대응**: 입력받은 파일명에서 경로 조작 문자열(`../`, `..\`)을 필터링하고, 업로드 경로를 안전한 특정 디렉토리로 고정함.
- **조치**: 사용자 입력 파일명을 그대로 사용하지 않고 서버측에서 생성한 고유한 식별자로 변경하여 저장하며, 업로드 디렉토리의 실행 권한을 제한함.
