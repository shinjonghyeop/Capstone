## Path Traversal
- **End-Point**: `/www/FileUpload/fileupload1.php`, `/www/FileInclusion/pages/lvl1.php` 외 1개
- **영향**: 공격자가 서버의 임의 디렉터리에 파일을 업로드하거나 기존 파일을 덮어쓸 수 있으며, 이는 시스템 권한 탈취나 원격 코드 실행(RCE)으로 이어질 수 있음.
- **설명**: `file` 파라미터를 통해 전달된 값이 `move_uploaded_file()` 함수에서 적절한 검증 없이 사용되어 경로 조작 공격이 발생할 수 있음.
- **근거**: `file` 파라미터를 대상으로 한 파일 업로드 재현 확인. `curl "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -e "http://192.168.64.2:9991/www/FileUpload/fileupload1.php" -F "submit=Submit" -F "file=@your_local_file;filename=fileupload1.php"`
- **대응**: 사용자로부터 입력받은 파일 이름에서 경로 조작 문자열(`../`, `..\\`)을 필터링하고, 파일 이름에 대한 유효성 검증을 수행해야 함.
- **조치**: 업로드된 파일의 이름을 서버에서 난수화된 고유 명칭으로 변경하여 저장하고, 업로드 디렉터리에 대한 실행 권한을 제거하며 접근 통제를 강화함.
