## Content Security Policy Configuration
- **End-Point**: `/cgi-bin/test.cgi`, `/text/css` 외 3개
- **영향**: Content Security Policy(CSP)가 설정되지 않은 경우 교차 사이트 스크립팅(XSS), 클릭재킹, 데이터 삽입 공격 등 악의적인 스크립트 실행 및 리소스 로드 공격에 노출될 위험이 있습니다.
- **설명**: `http://10.67.139.88/cgi-bin/test.cgi`를 포함한 해당 URL들에 브라우저 측 보안 정책인 CSP 헤더가 구성되어 있지 않아 보안 수준이 취약한 상태입니다.
- **근거**: `curl "http://10.67.139.88/cgi-bin/test.cgi"` 명령을 통한 재현 확인 결과, 응답 헤더에서 CSP 관련 설정이 확인되지 않음.
- **대응**: 웹 서버 설정 또는 애플리케이션 코드 내에서 `Content-Security-Policy` HTTP 응답 헤더를 정의하여 신뢰할 수 있는 도메인으로부터의 리소스만 허용하도록 구성해야 합니다.
- **조치**: 응답 헤더에 `Content-Security-Policy: default-src 'self';`와 같은 기본 정책을 추가하고, 서비스 요구사항에 따라 `script-src`, `style-src`, `img-src` 등의 지시문을 상세히 설정하십시오.
