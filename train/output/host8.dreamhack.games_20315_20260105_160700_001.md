## Content Security Policy Configuration
- **End-Point**: `/`, `/ping`
- **영향**: 콘텐츠 보안 정책(CSP)이 설정되지 않으면 사이트 간 스크립팅(XSS), 클릭재킹 및 데이터 주입 공격에 취약해지며, 공격자가 사용자의 브라우저에서 악성 스크립트를 실행하여 세션 정보나 민감한 데이터를 탈취할 수 있음.
- **설명**: 해당 웹 애플리케이션의 응답 헤더에 CSP가 설정되어 있지 않아, 브라우저가 신뢰할 수 있는 소스로부터 온 리소스인지를 판단할 수 있는 메커니즘이 부재함.
- **근거**: `curl "http://host8.dreamhack.games:20315/"` 명령을 통한 재현 시 응답 헤더에서 `Content-Security-Policy` 항목이 확인되지 않음.
- **대응**: 웹 서버 또는 애플리케이션 레벨에서 적절한 CSP 정책을 수립하여 HTTP 응답 헤더에 포함시켜야 하며, 허용된 출처 이외의 스크립트 및 리소스 로드를 차단해야 함.
- **조치**: 모든 응답에 `Content-Security-Policy: default-src 'self';`와 같이 엄격한 기본 정책을 적용하고, 서비스 운영에 필요한 외부 도메인이 있는 경우 해당 도메인만 선별적으로 허용하도록 설정함.
