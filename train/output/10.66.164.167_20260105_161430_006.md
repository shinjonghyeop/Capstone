## Unencrypted Channels
- **End-Point**: `/news.php`, `/dashboard.php` 외 1개
- **영향**: 암호화되지 않은 통신 채널을 통해 쿠키와 같은 민감한 정보가 노출되어 세션 탈취 및 중간자 공격(MITM)의 위험이 발생할 수 있음.
- **설명**: `http://10.66.164.167/news.php`로의 연결이 암호화되지 않은 HTTP를 통해 이루어지며, 응답에 포함된 쿠키 등 민감한 데이터가 노출됨. 서버가 리다이렉션을 수행하되 HTTPS로 전환하지 않아 보안 취약점이 유지됨.
- **근거**: `curl "http://10.66.164.167/news.php"` 실행 시 암호화되지 않은 채널을 통해 데이터가 전송되고 리다이렉션 시에도 평문 통신이 유지되는 것을 확인.
- **대응**: 서비스 전반에 SSL/TLS를 적용하여 HTTPS 통신을 강제하고, HSTS(HTTP Strict Transport Security)를 도입하여 보안 연결을 유지해야 함.
- **조치**: 웹 서버 설정에서 HTTP 요청을 HTTPS로 자동 리다이렉트하도록 구성하고, 쿠키 생성 시 `Secure` 속성을 추가하여 암호화된 연결에서만 전송되도록 설정함.
