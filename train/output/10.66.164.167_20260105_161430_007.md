## Inconsistent Redirection
- **End-Point**: `/news.php`, `/dashboard.php`
- **영향**: 리다이렉션 응답 본문에 민감한 정보나 실행 가능한 HTML 요소가 포함될 경우, 클라이언트가 리다이렉션을 무시하도록 설정되어 있다면 의도치 않은 데이터 노출이나 보안 위협이 발생할 수 있음.
- **설명**: 서버가 3xx 상태 코드를 반환하여 페이지 이동을 지시하면서도, 응답 본문에 링크나 폼과 같은 예상치 못한 HTML 데이터를 포함하는 취약점임.
- **근거**: `curl "http://10.66.164.167/news.php"` 실행 시 3xx 리다이렉션 상태 코드와 함께 의도하지 않은 HTML 본문이 반환됨을 확인.
- **대응**: 리다이렉션 헤더를 전송한 직후 서버 측 스크립트의 실행을 즉시 중단하여 추가적인 본문 데이터가 생성되지 않도록 처리해야 함.
- **조치**: PHP 소스 코드에서 `header("Location: ...")` 함수 호출 바로 다음에 `exit;` 또는 `die();` 구문을 추가하여 이후의 HTML 코드나 로직이 실행되지 않도록 수정함.
