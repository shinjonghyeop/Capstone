## MIME Type Confusion
- **End-Point**: `/`, `/download` 외 1개
- **영향**: 브라우저가 응답 본문의 내용을 분석하여 MIME 타입을 임의로 결정하는 MIME Sniffing을 통해 악성 스크립트가 실행될 수 있으며, 이는 Cross-Site Scripting(XSS) 공격으로 이어질 수 있음.
- **설명**: 서버 응답 헤더에 `X-Content-Type-Options`가 설정되어 있지 않아, 브라우저가 리소스의 MIME 타입을 잘못 해석할 가능성이 존재함.
- **근거**: `curl "http://192.168.64.2:5000/"` 실행 시 응답 헤더에서 `X-Content-Type-Options` 설정이 확인되지 않음.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정을 추가하여 브라우저가 선언된 Content-Type을 강제로 준수하도록 설정해야 함.
- **조치**: 웹 서버 또는 애플리케이션 설정 파일에서 `X-Content-Type-Options` 헤더 값을 `nosniff`로 적용함.
