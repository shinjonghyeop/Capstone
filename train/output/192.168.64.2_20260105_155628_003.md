## Django Debug Page - Cross-Site Scripting
- **End-Point**: `http://192.168.64.2:9991/www/XSS/XSS_level1.php/create_user/`
- **영향**: 공격자가 피해자의 브라우저에서 임의의 JavaScript 코드를 실행할 수 있으며, 이를 통해 세션 하이재킹, 웹사이트 변조, 민감한 정보 탈취 등이 가능함.
- **설명**: Django 1.10.x(1.10.8 미만) 및 1.11.x(1.11.5 미만) 버전의 기술적인 500 디버그 페이지 템플릿 일부에서 HTML 자동 이스케이프가 비활성화되어 발생하는 취약점(CVE-2017-12794)으로, `DEBUG=True` 설정 시 XSS 공격이 가능함.
- **근거**: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (SS; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/128.0.0.0 Safari/537.36' 'http://192.168.64.2:9991/www/XSS/XSS_level1.php/create_user/?username=%3Cscript%3Ealert(document.domain)%3C%2Fscript%3E'` 명령을 통해 `username` 파라미터에 삽입된 스크립트가 실행됨을 확인.
- **대응**: 운영 환경에서는 반드시 디버그 모드를 비활성화하고, 해당 취약점이 패치된 최신 버전의 Django로 업데이트해야 함.
- **조치**: `settings.py` 설정 파일에서 `DEBUG = False`로 변경하거나, Django 버전을 1.10.8 또는 1.11.5 이상으로 업그레이드함.
