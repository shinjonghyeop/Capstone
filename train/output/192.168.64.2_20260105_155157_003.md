## Django Debug Page - Cross-Site Scripting
- **End-Point**: `http://192.168.64.2:9991/www/XSS/XSS_level1.php/create_user/`
- **영향**: 공격자가 피해자의 브라우저에서 임의의 JavaScript 코드를 실행할 수 있으며, 이를 통해 세션 하이재킹, 웹사이트 변조 또는 민감한 정보 탈취가 가능합니다.
- **설명**: Django 1.10.x(1.10.8 미만) 및 1.11.x(1.11.5 미만) 버전의 500 에러 디버그 페이지 템플릿 일부에서 HTML 자동 이스케이프 기능이 비활성화되어 발생하는 취약점(CVE-2017-12794)입니다. `DEBUG=True` 설정 시 공격자가 조작된 페이로드를 통해 XSS 공격을 수행할 수 있습니다.
- **근거**: 다음 재현 명령을 통해 `username` 파라미터에 삽입된 스크립트가 실행됨을 확인하였습니다. `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:121.0) Gecko/20100101 Firefox/121.0' 'http://192.168.64.2:9991/www/XSS/XSS_level1.php/create_user/?username=%3Cscript%3Ealert(document.domain)%3C%2Fscript%3E'`
- **대응**: 운영 환경(Production)에서는 반드시 `settings.py`의 `DEBUG` 설정을 `False`로 변경하고, 취약점이 해결된 Django 최신 버전으로 업데이트해야 합니다.
- **조치**: Django 버전을 1.10.8 또는 1.11.5 이상으로 업그레이드하고, 서버 설정에서 `DEBUG = False`를 적용합니다.
