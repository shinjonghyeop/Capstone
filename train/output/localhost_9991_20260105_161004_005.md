## Django Debug Page - Cross-Site Scripting
- **End-Point**: `http://localhost:9991/www/XSS/XSS_level1.php/create_user/`
- **영향**: 공격자가 피해자의 브라우저 컨텍스트에서 임의의 JavaScript 코드를 실행할 수 있으며, 이를 통해 세션 하이재킹, 웹사이트 변조, 민감한 정보 탈취 등이 가능함.
- **설명**: Django 1.10.8 미만 및 1.11.5 미만 버전의 500 디버그 페이지 템플릿에서 HTML 자동 이스케이프가 비활성화되어 발생하는 취약점(CVE-2017-12794)임. `DEBUG=True` 설정 시 악의적인 스크립트가 포함된 요청을 통해 교차 사이트 스크립팅(XSS) 공격이 수행될 수 있음.
- **근거**: `curl -X 'GET' -d '' -H 'Accept: */*' -H 'Accept-Language: en' -H 'User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_8_2) AppleWebKit/536.26.17 (KHTML, like Gecko) Version/6.0.2 Safari/536.26.17' 'http://localhost:9991/www/XSS/XSS_level1.php/create_user/?username=%3Cscript%3Ealert(document.domain)%3C%2Fscript%3E'` 명령을 통해 `username` 파라미터에 삽입된 스크립트가 디버그 페이지에서 실행됨을 확인.
- **대응**: 운영 환경의 `settings.py` 파일에서 `DEBUG = False`로 설정하여 상세한 에러 페이지가 외부에 노출되지 않도록 구성함.
- **조치**: 취약점이 해결된 Django 버전(1.10.8 이상, 1.11.5 이상 또는 최신 안정 버전)으로 업그레이드를 수행함.
