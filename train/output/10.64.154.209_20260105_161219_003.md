## Spring Security BCrypt Truncation & ACL Bypass Detection
- **End-Point**: `http://10.64.154.209/cgi-bin/test.cgi/admin/search`
- **영향**: CVE-2025-22228 취약점을 악용한 인증 우회 및 관리자 권한 탈취로 인한 민감 정보 노출 가능성.
- **설명**: BCrypt 알고리즘의 72바이트 길이 제한 특성을 이용해 잘린 비밀번호로 로그인을 시도하고, X-Forwarded-For 헤더를 조작하여 접근 제어 목록(ACL)을 우회하는 공격임.
- **근거**: `curl -X 'POST' -d '{"apiKey": "test", "password": "aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaB"}' -H 'Content-Type: application/json' -H 'Host: 10.64.154.209' -H 'User-Agent: Mozilla/5.0 (SS; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/134.0.0.0 Safari/537.36' -H 'X-Forwarded-For: 127.0.0.1' 'http://10.64.154.209/cgi-bin/test.cgi/admin/search'` 명령을 통해 72자 이상의 비밀번호와 로컬 IP 헤더를 전송하여 재현함.
- **대응**: Spring Security를 해당 취약점이 해결된 최신 보안 패치 버전으로 업데이트하고, 신뢰할 수 없는 프록시 헤더 기반의 ACL 설정을 지양함.
- **조치**: 입력 단계에서 비밀번호 최대 길이를 검증하는 로직을 추가하고, 인프라 및 애플리케이션 설정에서 X-Forwarded-For 헤더의 신뢰 범위를 엄격히 제한함.
