## Reflected Cross Site Scripting
- **End-Point**: `/vuln`
- **영향**: 공격자가 사용자의 브라우저에서 악성 스크립트를 실행하여 세션 하이재킹, 중요 정보 탈취, 피싱 사이트 유도 등을 수행할 수 있습니다.
- **설명**: `param` 파라미터를 통해 입력받은 값이 적절한 필터링이나 인코딩 없이 응답 페이지에 포함되어 브라우저에서 스크립트가 실행되는 취약점입니다.
- **근거**: `param` 파라미터에 스크립트 주입 시 실행됨을 확인. 재현: `curl "http://127.0.0.1:8000/vuln?param=%3CScRiPt%3Ealert%28%27wup2qh6qxi%27%29%3C%2FsCrIpT%3E"`
- **대응**: 사용자 입력값에 대해 화이트리스트 기반의 검증을 수행하고, 웹 페이지 출력 시 반드시 HTML 엔티티 인코딩을 적용해야 합니다.
- **조치**: 응답 시 `param` 데이터를 `<` → `&lt;`, `>` → `&gt;` 등과 같이 인코딩하거나, `Content-Security-Policy` 설정을 통해 인라인 스크립트 실행을 차단합니다.
