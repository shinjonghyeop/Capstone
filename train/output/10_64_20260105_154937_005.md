## HttpOnly Flag cookie
- **End-Point**: `/index.php`
- **영향**: Cross-Site Scripting (XSS) 취약점 발생 시 공격자가 자바스크립트를 사용하여 사용자의 세션 쿠키(PHPSESSID)를 탈취할 수 있으며, 이는 세션 하이재킹으로 이어질 수 있음.
- **설명**: `http://10.64.141.227/index.php`에서 발급하는 `PHPSESSID` 쿠키에 `HttpOnly` 플래그가 설정되어 있지 않아 브라우저의 클라이언트 사이드 스크립트가 해당 쿠키에 접근 가능한 상태임.
- **근거**: `curl "http://10.64.141.227/index.php"` 실행 시 반환되는 응답 헤더의 `Set-Cookie` 설정에서 `HttpOnly` 속성이 누락된 것을 확인함.
- **대응**: 세션 관리 및 민감한 정보를 담고 있는 모든 쿠키 생성 시 `HttpOnly` 속성을 명시적으로 부여하여 클라이언트 사이드 스크립트(JavaScript)를 통한 접근을 차단해야 함.
- **조치**: PHP 설정 파일(`php.ini`)에서 `session.cookie_httponly = On`을 설정하거나, 애플리케이션 소스 코드에서 쿠키 생성 시 `HttpOnly` 플래그를 `true`로 설정함.
