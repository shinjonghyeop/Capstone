## MIME Type Confusion
- **End-Point**: `/admin/`, `/` 외 1개
- **영향**: 브라우저가 응답 본문의 내용을 추측하여 실행하는 MIME Sniffing 과정에서 공격자가 의도한 악성 스크립트가 실행될 수 있으며, 이는 교차 사이트 스크립팅(XSS) 등의 보안 위협으로 이어질 수 있음.
- **설명**: HTTP 응답 헤더에 `X-Content-Type-Options: nosniff` 설정이 누락되어 있어, 브라우저가 선언된 `Content-Type`과 다른 형식으로 데이터를 해석할 수 있는 취약점임.
- **근거**: `curl "http://10.67.167.75:8161/admin/"` 호출 시 응답 헤더에서 `X-Content-Type-Options` 설정이 확인되지 않음.
- **대응**: 모든 HTTP 응답 헤더에 `X-Content-Type-Options: nosniff`를 추가하여 브라우저가 MIME Sniffing을 수행하지 않고 선언된 타입으로만 처리하도록 강제해야 함.
- **조치**: 웹 서버(Apache, Nginx 등) 또는 애플리케이션 프레임워크 설정에서 해당 보안 헤더를 일괄 적용하도록 구성함.
